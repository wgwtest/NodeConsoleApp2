Level Runtime Probe

Local runtime probe for the level pack bridge. This page reloads DataManagerV2.js, resolves level_1_1, and verifies that runtime consumes the active level pack artifact instead of relying on editor memory state.

本页用途

运行时消费页本页是“运行时消费页”，会重新加载 DataManagerV2.js，再基于当前关卡 pack 解析 level_1_1，并验证运行时是否真的消费了 override 后的关卡配置。

它和 level_editor_io_test.html 的职责不同：I/O 页负责写入 override，本页负责在一个独立运行时里读出 override，并进一步实例化关卡敌人。

字段怎么读

  • Pack Source：`file` 表示走默认文件，`override` 表示运行时已经读到 override 存储。
  • Resolved Level Name：运行时真正解析出来的关卡名。
  • Resolved Enemy Members：运行时从关卡波次和敌人池映射出来的成员列表。
  • Instantiated Enemy Names：真正调用 instantiateLevel() 后实例化出来的敌人名。

你在本页应该看什么

  1. 点 Reload Runtime 后，页面会重新读取当前关卡 pack。
  2. 若前一页已写入 override，则 Pack Source 应显示为 override。
  3. Resolved Level Name 和 Resolved Enemy Members 应显示覆写后的样本值。
  4. 点 Instantiate Probe Level 后，Instantiated Enemy Names 应显示运行时实例化出的敌人名。

为什么这能证明

本页会重新加载 DataManagerV2.js，不共享 I/O 页内存。如果这里仍能读到 override 后的关卡名和敌人池成员，说明运行时消费的是“存储里的产物”而不是“上一页的内存对象”。再调用 instantiateLevel() 得到具体敌人名，就证明关卡 pack 到实例化链路已经贯通。

按钮说明

按钮 点击后会做什么 为什么这能证明
Reload Runtime 重新加载 DataManagerV2.js 和关卡 pack 解析结果。 确保当前显示的是一次全新的运行时解析结果，而不是旧页面残留。
Instantiate Probe Level 调用运行时的 instantiateLevel(level_1_1)，把关卡真正实例化成敌人对象。 证明 override 不只是“能显示”，而是能驱动真实实例化流程。
Open level_editor_io_test.html 回到产物注入页继续写入或清除 override。 用来和上一页配合，形成“写入 - 读取 - 实例化”的完整验证链路。
Open mock_ui_v11.html 打开主流程页面做进一步人工烟测。 用于在局部验证页确认 override 链路后，再回到真实主流程观察是否有副作用。
