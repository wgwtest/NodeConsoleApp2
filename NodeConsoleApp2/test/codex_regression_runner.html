Codex Regression Runner

本页用途

本页是自动回归页。它会在打开后自动执行全部回归用例，用来验证代码层、契约层、页面入口文案和关键链路没有被后续修改破坏。

本页会做什么

  • 自动运行已登记的测试函数。
  • 输出每一项的 PASS / FAIL。
  • 给出最小化的技术佐证，确认回归没有被打坏。

如何理解 PASS / FAIL

  • PASS：说明当前自动化覆盖到的契约仍成立。
  • FAIL：说明至少有一个既定契约或入口被破坏，需要先修复再继续验收。
  • 它证明的是“自动回归稳定”，不是“用户一定能自然理解页面”。

为什么它不能替代人工验收

本页只能检查预先写进测试里的技术契约，不能判断页面是不是容易理解、按钮语义是不是清楚、操作流程是不是符合你的真实认知。所以它只能当辅助证据，不能替代人工验收。

推荐用法

  1. 先在对应工具页做人工观察。
  2. 再打开本页，确认自动回归没有新增 FAIL。
  3. 人工结论仍然应该写回对应 WBS 验收清单。
