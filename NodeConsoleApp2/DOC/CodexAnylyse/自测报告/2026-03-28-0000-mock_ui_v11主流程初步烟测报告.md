## 4. 新发现的问题

### F-UI-01 页面存在重复引擎初始化

控制台出现两次：

- `Engine initializing...`

同时资源请求存在重复加载：

- `config.json`
- `player.json`
- `items.json`
- `enemies.json`
- `levels.json`
- `buffs_v2_7.json`
- `slot_layouts.json`
- `skills_melee_v4_5.json`

初步判断：

- `mock_ui_v11.html` 通过 `import { CoreEngine }` 手动实例化了一个引擎
- 但 `CoreEngine.js` 文件底部本身还会创建默认单例并挂到 `window.Engine`
- 导致页面可能同时存在：
  - 默认单例
  - 页面手动创建实例 `window.GameEngine`

### F-UI-02 关卡选择模态在进入战斗后未正确隐藏

现象：

- 状态已进入：
  - `BATTLE_LOOP`
  - `PLANNING`
- 但系统模态仍处于显示状态
- 页面可见标题仍然是：
  - `选择关卡`

这说明：

- `mock_ui_v11` 比旧入口更接近真实主 UI
- 但也暴露出 UI 层自己的宿主状态问题

## 5. 当前结论

当前结论应修正为：

- `mock_ui_v11.html` 才是主流程 UI 自测入口
- `engine_test_v2.html` 应降级为引擎辅助验证入口
- 上一轮深度自测报告不能直接作为主 UI 验收报告
