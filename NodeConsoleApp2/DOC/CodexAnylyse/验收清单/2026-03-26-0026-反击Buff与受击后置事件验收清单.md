## 1. 验收对象

本轮针对战斗主流程补齐两类关键缺口：

- 主伤害链路补入受击后置事件
  - `BATTLE_TAKE_DAMAGE`
  - `BATTLE_DEFEND_POST`
- `ATTACK` 类 Buff 从占位改为真实桥接

## 5. 分组验收项

### A. 代码边界

- [ ] 改动集中在引擎主链与测试页
- [ ] 没有扩散到无关 UI 或内容数据文件

失败判定：

- 改动扩散到无关模块
- 为了验证反击而改动正式内容数据

### B. 静态检查

建议命令：

```bash
npx eslint script/engine/CoreEngine.js \
  script/engine/EnemyActionPlanner.js \
  script/engine/DataManagerV2.js \
  script/engine/buff/BuffRegistry.js \
  script/engine/buff/BuffManager.js \
  script/engine/buff/BuffSystem.js
```

- [ ] 命令可执行
- [ ] 无阻塞级 error

失败判定：

- `eslint` 报错
- 新增桥接包含明显静态错误

### C. 事件链路

- [ ] `CoreEngine` 在伤害落地后 emit `BATTLE_TAKE_DAMAGE`
- [ ] `CoreEngine` 在防御结算后 emit `BATTLE_DEFEND_POST`
- [ ] `BuffSystem` 不再把 `ATTACK` 当成空实现
- [ ] `CoreEngine` 能接住 `BUFF_ATTACK_REQUEST`

失败判定：

- 事件仍未发出
- 反击仍只存在于数据和日志描述层

### D. 浏览器最小实战

#### 对照组 1：不挂反击 Buff

- [ ] 进入 `level_1_1`
- [ ] 空规划执行后进入第 2 回合
- [ ] 玩家 HP 下降
- [ ] 敌人 HP 保持不变
- [ ] 日志中无 `triggered 反击`

失败判定：

- 敌人异常掉血
- 无法进入第 2 回合

#### 对照组 2：挂 `buff_counter`

- [ ] 进入 `level_1_1`
- [ ] 给玩家挂 `buff_counter`
- [ ] 空规划执行后进入第 2 回合
- [ ] 敌人 HP 低于对照组
- [ ] 日志出现 `triggered 反击`

失败判定：

- 挂 Buff 后敌人 HP 与对照组相同
- 有 Buff 但没有反击日志
