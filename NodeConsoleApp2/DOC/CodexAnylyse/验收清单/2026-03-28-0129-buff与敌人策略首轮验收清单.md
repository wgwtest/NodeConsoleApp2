## 4. 分组验收项

### A. Buff 链路

- [ ] `AP_COST_REDUCE` 在 `TURN_START` 后正确写入 `-1` 规划开销修正
- [ ] `buff_counter` 在 `BATTLE_DEFEND_POST` 后发出 `BUFF_ATTACK_REQUEST`
- [ ] 反击请求中的 `source` 和 `target` 方向正确
- [ ] `PREVENT_DAMAGE_HP` 在真实伤害管线中阻止 HP 损失
- [ ] `PREVENT_DAMAGE_ARMOR` 在真实伤害管线中阻止护甲消耗
- [ ] `PREVENT_DAMAGE_ARMOR` 生效时伤害不会直穿到 HP
- [ ] `HEAL_ARMOR` 可在受击前补甲，并由新增护甲吸收当前伤害

### B. 敌人策略

- [ ] 受损敌人不再固定选择攻击技能
- [ ] 受损敌人会切换到 `skill_block`
- [ ] 修甲目标部位指向最破损部位 `chest`
- [ ] 低血量敌人会切换到 `skill_recover`
- [ ] 低血量回血技能的目标仍指向敌人自身

### C. 回归基线

- [ ] 新增断言后自动回归全通过
- [ ] 本轮改动未破坏既有主流程回归项
- [ ] `mock_ui_v11.html` 中多选技能可部署、提交并执行到下一回合

## 5. 失败判定点

出现以下任一情况，可判定本轮不通过：

1. 敌人胸部重伤时仍选择 `skill_bite`
2. `buff_counter` 不发出反击请求
3. 降耗 Buff 不修改规划开销
4. `PREVENT_DAMAGE_HP` 触发后 HP 仍下降
5. `PREVENT_DAMAGE_ARMOR` 触发后伤害直穿到 HP
6. 受击获甲后当前伤害未被护甲吸收
7. 新增断言导致既有主流程回归项转红
8. `mock_ui_v11.html` 的多选技能主流程无法进入下一回合

## 6. 建议验收顺序

1. 先跑 `codex_regression_runner.html`
2. 确认 12 项断言全通过
3. 再走一次 `mock_ui_v11.html` 多选技能抽样流程
4. 最后阅读对应自测报告
