# 引擎主流程改造第一阶段记录

创建时间：2026-03-25 23:53

## 1. 本轮目标

将 `NodeConsoleApp2` 的战斗引擎从“设计原型 + 旧 schema 兼容残留”推进到“新 schema 已进入主流程执行”的状态。

本轮明确目标：

- 技能执行层接入 `actions[].effect`
- Buff wrapped 数据接入运行时
- Buff 执行器兼容当前数据格式
- 敌人行为从占位逻辑升级为规则规划
- 接入行动前拦截与 AP 生命周期

本轮明确非目标：

- 完整 UI 重构
- 所有 Buff action 全量支持
- 完整反击系统
- 正式内容数据补全

## 2. 已完成项

### 2.1 技能执行主链改造

已完成：

- `CoreEngine` 不再以旧字段 `type/value/targetType` 为主执行入口
- 技能按 `actions[].effect` 执行
- 支持的主要 effectType：
  - `DMG_HP`
  - `DMG_ARMOR`
  - `HEAL`
  - `ARMOR_ADD`
  - `AP_GAIN`
  - `BUFF_REMOVE`
- `buffRefs.apply/remove` 已接入真实执行链路

### 2.2 Buff 主链改造

已完成：

- `DataManagerV2` 会从 `buffs_v2_7.json` 中正确拆出 `buffs`
- `BuffRegistry` 支持参数模板替换
- `BuffManager` 支持数组型 `statModifiers`
- `BuffSystem` 支持：
  - 大写 action 名称
  - `payload.value/valueType`
  - `HEAL_ARMOR`
  - `PREVENT_DAMAGE_HP`
  - `PREVENT_DAMAGE_ARMOR`
  - `AP_COST_ADD`
  - `AP_COST_REDUCE`
  - `SKIP_TURN`

### 2.3 敌人行为

已完成：

- 新增独立模块 `EnemyActionPlanner.js`
- 敌人不再固定取第一个技能
- 会基于：
  - 技能效果摘要
  - 自身血量
  - 玩家护甲/弱点
  - 技能目标类型
  - AP 可用性
  进行规则化选择

### 2.4 AP 与行动控制

已完成：

- `BATTLE_ACTION_PRE` 已接入真实动作执行前
- 被取消的动作不会误扣 AP
- 玩家每回合 AP 重置
- 敌人每回合 AP 重置
- 敌人执行技能时消耗 AP
