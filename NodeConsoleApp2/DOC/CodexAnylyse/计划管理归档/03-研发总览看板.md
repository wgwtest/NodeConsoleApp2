## 2.1 如何理解这张图

这张图表达的是“阶段推进顺序 + 阶段贡献到 Gate”，不是“所有卡片完全同级并列”。

正确理解方式：

1. `阶段` 是工作包
2. `Gate` 是验收节点，不是树节点
3. `当前执行切片` 不直接画在这张总图里，但它属于 `阶段 A`
4. `阶段 A` 服务于 `M1`
5. `阶段 B` 服务于 `M2`
6. `阶段 C` 服务于 `M3`
7. `阶段 D` 和 `阶段 E` 共同服务于 `M4`

Gate 是累计成立的：

1. `M1 = A`
2. `M2 = A + B`
3. `M3 = A + B + C`
4. `M4 = A + B + C +  D + E`
