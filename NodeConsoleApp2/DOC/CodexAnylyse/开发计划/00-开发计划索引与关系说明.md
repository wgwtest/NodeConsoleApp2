## 5. 后续维护规则

从现在开始，计划类文档统一放在：

- `DOC/CodexAnylyse/开发计划/`

新增计划时遵守以下规则：

1. 总计划只保留 1 份，持续演进，不再重复新建并列“总计划”
2. 当前执行计划只保留 1 份，切阶段时更新或替换
3. 过期阶段计划移入 `历史计划/`
4. 新计划必须在开头写清楚：
   - 对应总计划哪个阶段
   - 对应哪些 WBS 节点
   - 与上一份计划的关系
5. 如果涉及 GitHub 执行结构，必须同时说明：
   - 对应哪一层 WBS
   - 是否已经落地到 issue tree
   - 对应迁移记录是哪一份
