# 基于 mock_ui_v11 深测结论的后续研发计划

创建时间：2026-03-28 00:15

状态：`待用户审定`

## 1. 计划目标

基于本轮深测结论，把当前工程从“主流程可跑”推进到“今晚可形成一轮稳定可回归的主流程底座”，并为后续技能 / Buff / 敌人行为继续扩展留出清晰边界。

## 2. 计划边界

本计划优先处理今晚就应该收敛的问题：

1. 引擎实例唯一性
2. UI 宿主状态一致性
3. 技能目标契约到规划 UI 的一致性
4. 战斗内保存一致性
5. 多选技能执行一致性
6. 敌人行为主链路回归

今晚不承诺直接闭环的内容：

- 所有 Buff 分支的完整覆盖
- 所有敌人策略类型的完整覆盖
- 设置页体验与非主流程视觉问题

原因：

- 当前已经确认双引擎、模态残留、战斗存档、以及多选技能执行都有底层不一致
- 在这些问题未收敛前，直接承诺“今晚完成全部 Buff / 敌人行为闭环”不具备验证基础

## 3. 今晚压缩版研发节奏

建议改为今晚 4 个连续阶段推进，阶段之间即时回归，不拖到明天再集中验证。

### 阶段 A：稳定测试宿主与引擎实例

目标：

- 消除 `mock_ui_v11.html` 双引擎初始化
- 保证 UI 全部绑定到唯一运行时实例
- 重新验证数据资源不再重复加载

实施重点：

- 明确 `CoreEngine` 的页面接入方式，只保留一种实例化路径
- 明确 `window.Engine` / `window.GameEngine` 的唯一来源
- 重跑 `mock_ui_v11.html` 登录到回合 1 的主流程烟测

完成判定：

- 控制台不再成对出现 `Engine initializing...`
- 数据资源不再双份请求
- 主流程仍可进入回合 1

### 阶段 B：修复模态生命周期与焦点管理

目标：

- 消除选关模态战斗残留
- 修复技能树关闭后的焦点告警

实施重点：

- 统一系统模态显示 / 隐藏时机
- 关闭技能树时主动转移焦点到安全宿主节点
- 补一轮模态切换回归：登录、主菜单、选关、进战斗、打开技能树、关闭技能树

完成判定：

- 进入战斗后看不到 `选择关卡` 残留
- 控制台不再出现 `aria-hidden` 焦点告警

### 阶段 C：统一技能目标契约、保存入口和多选执行

目标：

- 让单选 / 多选技能都严格按契约高亮和放置
- 解决技能树学习后写出陈旧战斗快照的问题
- 修复多选技能“规划 4 个动作，但执行疑似不完整”的问题

实施重点：

- 让 `UI_SkillPanel` 消费 `candidateParts` / `selectedParts`
- 统一所有战斗中保存路径，禁止 UI 直接绕过引擎保存战斗态
- 对多选技能的规划动作、时间线装载、执行日志、历史记录逐段核对

完成判定：

- `裂颅一击` 只高亮头部目标槽
- `多攻测试` 可稳定放置 4 个槽位并完整执行 4 个动作
- 技能树学习后 `save_game` 的 `runtime.turn / runtime.phase / levelData` 与页面实时状态一致

### 阶段 D：今晚收口回归

目标：

- 在修复后的底座上，重新跑通主流程、单选技能、多选技能、技能树、敌人行为
- 产出今晚的实现验收材料

实施重点：

- 回归主流程 `开始冒险 -> 关卡选择 -> 选关 -> 规划 -> 执行`
- 回归单选技能 `重锤 / 裂颅一击`
- 回归多选技能 `多攻测试`
- 回归敌人自动行为 `skill_bite`
- 形成新的自测报告、验收清单、交接记录

完成判定：

- 至少 1 条单选技能链路通过
- 至少 1 条多选技能链路通过
- 敌人自动行为主链路通过
- 自测材料可直接交你验收

## 4. 风险与依赖

### 高风险项

- 双引擎问题如果不先处理，后续任何 UI / Buff / 敌人行为结论都可能失真
- 战斗内存档不统一会污染回归样本
- 多选技能如果继续存在“预览 4 个动作、执行不足 4 个”的问题，主流程战斗结论仍不可靠

### 中风险项

- 技能新 schema 与旧 UI 读取逻辑存在字段脱节
- 模态宿主和页面主视图可能存在更多状态耦合点

## 5. 建议验收方式

建议按今晚压缩版推进并验收：

1. 先批准今晚的 4 个连续阶段
2. 我完成今晚阶段后，直接给你新的：
   - 自测报告
   - 验收清单
   - 会话交接记录
3. 你验收通过后，再决定是否立即进入 Buff / 敌人行为扩展轮

## 6. 我对本计划可行性的判断

压缩到今晚是可行的，但前提是严格遵守以下顺序：

- 先做宿主与状态一致性，再做技能契约与多选执行
- 不在双引擎未收敛前继续扩展 Buff 分支
- 今晚的目标定义为“主流程底座闭环”，不是“全项目所有行为闭环”

如果你批准，我会按这个今晚版计划直接进入实现，而不是继续只做文档。
