## 1. 本文目的

本文不直接修改技能、敌人、关卡或 KP 数值。

本文用于回答一个更上层的问题：

> `NodeConsoleApp2` 的战斗与成长到底应该让玩家爽在哪里？

当前项目已经具备技能树、AP、部位护甲、敌人技能、关卡奖励和地图选择等系统外形，但这些系统还没有稳定汇聚成一个清晰的核心体验。

玩家当前反馈集中在：

1. 技能成长性不强。
2. KP 点数过多，学习没有压力。
3. 学两个攻击技能就能过关，敌人没有逼出构筑选择。
4. 攻击、防御、AP 花活都存在，但没有形成流派承诺。
5. 找不到游戏的核心痛点，也就找不到核心爽点。

本文的目标是把这些问题转成可讨论、可验证、可落地的设计判断。

## 2. 当前核心问题判断

### 2.1 问题不是技能数量不足

当前玩家技能数量已经不少，敌人数量和关卡数量也已经扩充到可试玩规模。

如果继续直接新增技能，很容易出现以下结果：

1. 技能按钮更多，但玩家仍然只选最高伤害技能。
2. 技能树更复杂，但学习顺序没有真正代价。
3. 敌人更多，但每个敌人只是血量、护甲、状态的排列组合。
4. 自动试玩可能更容易通过，但玩家仍然没有构筑快感。

因此当前阶段的重点不应是继续堆内容，而应先确认：

1. 玩家为什么必须做选择。
2. 玩家做了选择以后，战斗方式如何改变。
3. 敌人如何验证玩家选择是否有效。
4. 奖励和 KP 如何制造长期取舍。

### 2.2 KP 经济抹平了构筑压力

当前默认玩家拥有较多 KP，而大量技能成本集中在 1、2、3 KP。

这会造成一个直接问题：

> 玩家不是在构筑，而是在购物。

构筑快感来自稀缺性。玩家必须感觉到：

1. 这次只能学一个方向。
2. 学攻击就暂时牺牲防御。
3. 学防御就暂时缺爆发。
4. 学 AP 花活就需要承担更高操作和代价。
5. 现在的选择会影响下一关的体验。

如果默认 KP 足够买下大量基础技能，那么技能树会失去“路线选择”的意义。

### 2.3 攻击技能目前过于接近默认最优解

当前低 AP 攻击技能能快速解决多数早期敌人。

当敌人没有明确提出问题时，玩家自然会选择最简单的解法：

1. 选中攻击技能。
2. 放到敌人可打部位。
3. 提交规划。
4. 重复直到胜利。

这种流程本身可以成立，但不能长期作为游戏核心。

如果攻击技能没有遇到护甲、修复、反击、状态、速度和部位压力，那么攻击成长只会变成线性伤害提升，而不是流派成长。

### 2.4 防御和 AP 花活缺少被需要的场景

防御技能的价值不是“能补甲”，而是“敌人让我不得不补甲”。

AP 技能的价值不是“能多动”，而是“我能为了爆发窗口承担代价”。

如果敌人压力不足，防御和 AP 花活都会变成可有可无：

1. 不防御也能打过，防御就是拖慢胜利。
2. 不做 AP 操作也能打过，AP 花活就是复杂但不必要。
3. 不处理状态也能打过，净化、续航和节奏技能就没有舞台。

所以技能成长不强，根因不只在技能本身，也在敌人没有稳定制造“必须回答的问题”。

## 3. 推荐核心体验

### 3.1 一句话核心

推荐把本游戏的核心体验定义为：

> 在有限 KP、有限 AP、有限部位槽位下，识别敌人的压力点，用构筑把压力转化为自己的爆发窗口。

这句话包含四个关键约束：

1. `有限 KP`
   - 长期成长不能全都要。
2. `有限 AP`
   - 单回合不能什么都做。
3. `有限部位槽位`
   - 技能不是只消耗 AP，也占用身体和行动结构。
4. `敌人压力点`
   - 关卡必须提出明确问题，否则构筑没有意义。

也包含一个核心爽点：

> 玩家不是单纯堆数值，而是看懂局面、准备答案、打出窗口。

### 3.2 单局爽点结构

一场理想战斗应形成以下节奏：

1. **识别压力**
   - 敌人正在打弱点部位、叠毒、补甲、蓄力、吸血或拖时间。
2. **规划答案**
   - 玩家决定本回合是输出、补甲、破甲、解状态、抢速度，还是为下一回合攒爆发。
3. **承担代价**
   - 每个选择都占用 AP、部位槽、血量、护甲或后续机会。
4. **打出窗口**
   - 玩家成功破甲、压血、触发流血、挡住爆发或获得额外 AP。
5. **获得反馈**
   - 敌人血量、护甲、状态和行动顺序产生可见变化，让玩家确认构筑有效。

### 3.3 长线爽点结构

长期成长应形成以下循环：

```text
通关获得少量 KP
  -> 只能选择一个成长方向
  -> 下一关敌人验证该方向
  -> 如果选择有效，战斗方式明显变化
  -> 如果选择不匹配，玩家承受损耗或失败
  -> 回到技能树重新规划
```

这个循环的重点不是惩罚玩家，而是让玩家相信：

1. 我刚才的选择真的影响了下一场战斗。
2. 失败不是纯数值不够，而是我没有回答敌人的问题。
3. 新技能不是收藏品，而是改变打法的工具。

## 4. 三条主流派建议

当前可先收束成三条大流派，不建议一开始就拆太多。

### 4.1 攻击流：从稳定输出到爆发处决

#### 定位

攻击流不是简单提高伤害，而是围绕“如何把敌人打进可收割窗口”成长。

#### 成长阶段

| 阶段 | 玩家能力 | 敌人问题 | 爽点 |
| --- | --- | --- | --- |
| 初期 | 稳定直伤、指定部位攻击 | 低甲快攻敌 | 我能快速结束战斗 |
| 中期 | 破甲、弱点打击、DOT 前置 | 护甲敌、修复敌 | 我能先拆防线再杀人 |
| 后期 | 爆发、处决、连击、斩杀条件 | Boss、精英门卫 | 我能抓窗口一回合收掉 |

#### 设计要求

攻击流必须被以下敌人限制：

1. 高护甲敌人让纯直伤效率下降。
2. 修复敌人让低爆发输出被拖住。
3. 反击敌人让无脑多段攻击有风险。
4. 高速敌人让慢速大招可能来不及出手。

#### 不应出现

1. 低 AP、无条件、高伤害、无代价的通用攻击技能。
2. 学两个攻击技能即可无脑覆盖多数关卡。
3. 攻击技能只在数值上升级，不改变目标、窗口或代价。

### 4.2 防御流：从补甲续命到防反取胜

#### 定位

防御流不是“拖回合”，而是把敌人的攻击压力转成反击资源。

#### 成长阶段

| 阶段 | 玩家能力 | 敌人问题 | 爽点 |
| --- | --- | --- | --- |
| 初期 | 指定部位补甲、基础治疗 | 快攻、弱点追击 | 我能顶住第一轮压力 |
| 中期 | 受击获甲、减伤、状态处理 | DOT、重击、破甲 | 我能把敌人攻势吃下来 |
| 后期 | 防反、护甲转伤害、低血反打 | Boss 多段压迫 | 你打我越狠，我反打越强 |

#### 设计要求

防御流必须拥有明确收益：

1. 防住关键部位后，下一回合能换来输出窗口。
2. 护甲不只是血条外壳，而是能参与反击、转伤害或触发 Buff。
3. 防御技能必须有机会成本，不能变成每回合无脑必点。

#### 不应出现

1. 防御技能只延长战斗，没有反击或节奏收益。
2. 敌人伤害太低，导致防御技能没必要。
3. 敌人 DOT 太强，但玩家没有早期反制，导致防御流变成慢性死亡。

### 4.3 AP / 节奏流：从省 AP 到代价爆发

#### 定位

AP / 节奏流的核心不是“多放技能”，而是通过代价、速度和部位限制，制造爆发回合。

#### 成长阶段

| 阶段 | 玩家能力 | 敌人问题 | 爽点 |
| --- | --- | --- | --- |
| 初期 | 0-1 AP 填充、轻微加速 | AP 不够、槽位有限 | 我能多塞一个关键动作 |
| 中期 | 自伤换 AP、护甲换 AP、延迟换 AP | 修复敌、护甲敌 | 我能赌一轮爆发 |
| 后期 | 连锁、超载、回合压缩 | Boss 阶段、精英门卫 | 我能设计一个爆发回合 |

#### 设计要求

AP 技能必须有真实代价：

1. 扣 HP。
2. 扣某部位护甲。
3. 降低速度。
4. 锁定某部位槽位。
5. 下回合负面状态。
6. 只能在低血、破甲、叠层等窗口使用。

#### 不应出现

1. 无条件白给 AP。
2. AP 技能收益不明显，只是多一个按钮。
3. AP 连锁不需要计划，随便点也能赚。

## 5. 敌人压力如何服务爽点

敌人不是为了“变难”而存在，而是为了提出构筑问题。

### 5.1 敌人压力类型

| 压力类型 | 逼迫玩家回答的问题 | 推荐反制 |
| --- | --- | --- |
| 快攻弱点 | 我能否保护关键部位并快速结束？ | 补甲、治疗、低 AP 输出 |
| 高护甲 | 我能否破甲或绕过护甲？ | 破甲、DOT、处决窗口 |
| 修复续航 | 我能否阻止敌人拖长战斗？ | 爆发、DOT、打断、压制 |
| DOT 状态 | 我能否处理长期损耗？ | 治疗、净化、速杀 |
| 减速 / 晕眩 | 我能否管理行动顺序？ | 加速、控制反制、防御 |
| 反击 / 吸血 | 我能否选择正确爆发窗口？ | 控制、爆发前置、防御反打 |
| Boss 阶段 | 我能否综合使用本章学到的答案？ | 流派核心连锁 |

### 5.2 章节压力递进

建议三章压力不是单纯加数值，而是逐步增加问题复杂度。

| 章节 | 主压力 | 构筑教学 |
| --- | --- | --- |
| 第一章 | 快攻、弱点、基础护甲 | 让玩家理解部位、防御、低 AP 输出 |
| 第二章 | 护甲、修复、减速 | 让玩家理解破甲、节奏、持续输出 |
| 第三章 | DOT、吸血、组合压力、Boss 阶段 | 让玩家理解流派连锁和爆发窗口 |

### 5.3 每个关卡应有一句压力说明

每个正式关卡都应能回答：

> 这一关在验证玩家哪一种构筑能力？

例如：

1. `1-1` 验证：玩家是否知道攻击和补甲都能落槽。
2. `1-4` 验证：玩家是否第一次遇到不能只靠低伤平推的护甲敌。
3. `1-10` 验证：玩家是否能在 Boss 重击前防住关键部位。
4. `2-5` 验证：玩家是否能处理护甲修复。
5. `3-8` 验证：玩家是否能处理持续状态压力。
6. `3-10` 验证：玩家是否能打出完整流派窗口。

如果某关无法说清楚验证目标，它就很可能只是内容填充。

## 6. KP 与成长节奏

### 6.1 KP 的设计职责

KP 不只是奖励数字，而是构筑选择的稀缺资源。

KP 应承担三件事：

1. 限制玩家不能全都学。
2. 诱导玩家做流派承诺。
3. 让每次通关后的学习选择影响下一关。

### 6.2 推荐体验

早期玩家应该经常面对类似选择：

1. 我是学一个新攻击，还是先补防御？
2. 我是学破甲，为后面护甲敌做准备，还是先学治疗保证容错？
3. 我是走 AP 爆发，但接受更高风险，还是走稳定输出？

### 6.3 当前问题

当前默认 KP 太多，会导致：

1. 玩家一开始就能买大量技能。
2. 技能树前置失去意义。
3. 关卡奖励不再令人期待。
4. 技能设计里的短板被“我全都学”抹掉。

### 6.4 推荐原则

后续进入实现前，建议先确认以下原则：

1. 正式新档默认 KP 应接近 0，而不是调试用高 KP。
2. 第一章总 KP 应只足够支撑一个主方向和少量补点。
3. 关键流派技能应放在互斥或高机会成本位置。
4. Boss 前应该允许玩家形成一个明确流派，但不应全流派成型。
5. 支线 / 补给节点可以给临时资源，而不是长期 KP 泛滥。

## 7. 技能设计判断标准

后续每个技能进入正式技能树前，都应回答以下问题。

### 7.1 必答问题

1. 它属于攻击、防御、AP / 节奏，还是跨流派桥接？
2. 它解决哪个敌人压力？
3. 它要求玩家付出什么代价？
4. 它和哪些技能形成连锁？
5. 它在哪个章节第一次变得必要？
6. 它如果被玩家无脑使用，会有什么短板？

### 7.2 技能合格标准

一个合格技能至少应满足以下一条：

1. 改变目标选择。
2. 改变行动顺序。
3. 改变资源结构。
4. 提供新的反制手段。
5. 创造爆发窗口。
6. 把敌人压力转化成玩家收益。

如果一个技能只是“同样目标上多造成一些伤害”，它不应成为正式成长节点，除非它承担非常明确的基础教学职责。

### 7.3 技能不合格信号

以下技能设计应谨慎进入正式树：

1. 低 AP、高伤害、无条件、无副作用。
2. 和已有技能只有数值差异。
3. 描述看起来很酷，但战斗方式不变。
4. 依赖复杂 Buff，但 UI 无法解释。
5. 玩家不需要理解敌人也能无脑使用。

## 8. 当前项目的方向性结论

### 8.1 不应继续优先做的事

1. 不应继续新增无条件攻击技能。
2. 不应继续用默认 30 KP 做正式体验判断。
3. 不应把自动试玩全通当成唯一目标。
4. 不应在流派不清楚时反复微调单个敌技数值。
5. 不应把“技能能释放”误判为“技能有成长价值”。

### 8.2 应优先做的事

1. 固定三条主流派：攻击、防御、AP / 节奏。
2. 为每条流派设计 3 个成长阶段：初期、中期、后期。
3. 为每章定义明确敌人压力主题。
4. 重新规划 KP 稀缺曲线。
5. 把每个关卡标注为“验证哪种构筑能力”。
6. 把现有技能重新归类，剔除只有数值差异的节点。

### 8.3 核心体验目标

后续版本的玩家体验应从：

> 我学了两个攻击技能，所以一路打过去。

转变为：

> 我看出下一关是护甲 / 毒 / 快攻 / 修复压力，所以我这次选择了对应流派，并在战斗里打出了预期窗口。

这是构筑快感成立的最低标准。

## 9. 可验证的设计验收口径

后续任何实现或调参，都应能用以下问题验收：

1. 新档玩家在第一章前半段是否存在明确 KP 取舍？
2. 玩家只学攻击技能时，是否会在某些关卡明显承压？
3. 玩家走防御流时，是否能用防御转化出反击或续航收益？
4. 玩家走 AP / 节奏流时，是否能打出明显爆发回合，同时承担代价？
5. 每章 Boss 是否在检查本章核心压力，而不是单纯血厚？
6. 每个新增技能是否能说明它解决哪个敌人问题？
7. 每个新增敌人是否能说明它逼迫玩家使用哪类反制？
8. 自动试玩报告是否能区分“数值太高”和“玩家没有反制工具”？

如果这些问题无法回答，即使测试通过，也只能说明系统能运行，不能说明核心体验成立。

## 10. 下一步建议

建议下一步不要直接改代码，而是先做一张 `构筑闭环矩阵`：

| 维度 | 内容 |
| --- | --- |
| 玩家流派 | 攻击、防御、AP / 节奏 |
| 流派阶段 | 初期、中期、后期 |
| 代表技能 | 每阶段 2-4 个核心技能 |
| 敌人压力 | 快攻、护甲、修复、DOT、控制、Boss 阶段 |
| 关卡验证点 | 每个编号关卡验证什么 |
| KP 节奏 | 每章能学多少、何时形成流派 |
| 失败诊断 | 打不过时应补技能、补敌人、调奖励还是调数值 |

只有这张矩阵成立后，再进入具体的技能重排、KP 调整、敌人压力调整和自动试玩验证。
