# S6 敌人系统与编辑器设计说明

最后整理时间：2026-05-31

状态：`当前有效`

## 1. 子系统范围

本子系统负责敌人模板、敌人种族与职业、敌人属性与部位护甲、敌人技能组、敌人 AI 设计语义、敌人模板编辑器，以及与关卡编辑器、地图编辑器之间的 `templateId` 交接边界。

敌人血量、护甲、技能压力和关卡目标轮数必须优先遵循总纲中的 [战斗数值基准模型](../00_总纲/03-战斗数值基准模型(combat_numeric_baseline)-设计说明.md)。

## 3. 文档治理

敌人设计只保留一个主设计文档。后续新增敌人机制、流程压力、数值口径、反制矩阵、迁移计划和 V2 字段时，优先补充 `11-敌人系统(enemy_design)-设计说明.md`。

不要再新增平行的敌人机制说明、敌人迁移表或敌人平衡矩阵文档。若某项内容过长，先在主文档中保留摘要和索引，再评估是否需要独立附录。

## 4. 归属规则

1. 敌人模板字段、种族、职业、属性、部位护甲、技能组和 AI 行为语义归入本子系统。
2. 敌人编辑器归入本子系统，作为敌人模板内容生产工具。
3. 敌人技能定义归入 `S4` 的 skill schema 与技能编辑器能力；敌人模板只保存 `skills[]` 引用和 AI 语义。
4. 关卡编辑器与地图编辑器只消费敌人模板目录、摘要和 `templateId`，不直接维护敌人模板真值。
5. 战斗运行时如何实例化敌人、推进回合和执行行动归入 `S2_战斗运行时`；本子系统定义敌人内容语义与作者工具边界。
6. 跨模块数据装载、存档兼容和通用内容契约归入 `S7_数据存档与内容契约`。
