# WBS-4.3.7 动画参数配置器与展示配置资产化工具

记录时间：2026-04-11 10:09:01 +0800

状态：`已实现 / 已自测 / 待人工验收`

对应 WBS：`#97 / WBS-4.3.7 动画参数配置器与展示配置资产化工具`

## 1. 本节点解决的问题

在 `WBS-4.3.6` 之前，对战展示层虽然已有动作模板和节奏反馈，但关键参数仍然散落在：

1. `mock_ui_v11.css` 的写死动画常量
2. `BattleAnimationDriver / FighterPresenter` 的定值时长
3. `battle_presentation_probe.html` 的静态说明

这会带来三个问题：

1. 无法独立编辑和保存展示参数
2. probe 页与主流程页无法稳定共享同一份展示配置
3. 后续 `SpineAssets` 接入时缺少稳定的展示参数桥接面

## 2. 本轮实现

### 2.1 展示配置资产

新增：

1. `assets/data/battle_presentation_profiles_v1.json`

结构：

1. `schemaVersion`
2. `meta`
3. `activeProfileId`
4. `profiles[]`

当前提供两个 profile：

1. `default_balanced`
2. `impact_heavy`

### 2.2 配置装载层

新增：

1. `script/ui/presentation/BattlePresentationAssetStore.js`

提供能力：

1. 读取默认资产
2. 优先读取浏览器 workspace
3. 清空 workspace
4. 归一化 profile 文档
5. 生成 CSS 变量桥接表

### 2.3 展示消费层

更新：

1. `script/ui/presentation/BattlePresentationController.js`
2. `script/ui/presentation/BattleAnimationDriver.js`
3. `script/ui/presentation/FighterPresenter.js`
4. `script/ui/UI_BattleRow.js`
5. `mock_ui_v11.css`

关键变化：

1. `BattlePresentationController.applyPresentationProfile()` 负责把 active profile 写入：
   - scene dataset
   - CSS 变量
   - 主流程 / probe 读数节点
2. `BattleAnimationDriver / FighterPresenter` 通过 profile 读取：
   - 动画时长
   - 飘字时长
   - 场景脉冲时长
3. `mock_ui_v11.html` 与 `battle_presentation_probe.html` 都会显示当前演出配置资产来源

### 2.4 独立工具入口

新增：

1. `test/battle_presentation_configurator.html`

页面职责：

1. 明确说明“这是展示参数工具，不改战斗规则”
2. 编辑 active profile 的关键参数
3. 预览前冲 / 受击 / 治疗 / Buff / 镜头冲击
4. 写入浏览器 workspace
5. 从 workspace 读取
6. 恢复资产默认
7. 导出 / 导入 JSON

## 3. 当前桥接边界

当前桥接面固定为：

1. 主流程页：
   - `mock_ui_v11.html`
2. probe 页：
   - `test/battle_presentation_probe.html`
3. 配置器：
   - `test/battle_presentation_configurator.html`
4. 资产层：
   - `assets/data/battle_presentation_profiles_v1.json`
5. 装载层：
   - `script/ui/presentation/BattlePresentationAssetStore.js`

边界规则：

1. 规则层不写展示参数
2. 配置器不写伤害、Buff 或敌人行为
3. 主流程只消费 active profile
4. `SpineAssets` 后续只需对齐 profile / bridge 语义，不应反向侵入规则层
