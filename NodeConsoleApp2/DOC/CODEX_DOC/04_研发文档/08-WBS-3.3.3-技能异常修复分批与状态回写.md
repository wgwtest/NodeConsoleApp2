## 1. 文档职责

本文件不替代首轮异常归档，也不替代验收清单。

它的职责只有三件事：

1. 把 `issue code` 映射成正式修复批次；
2. 明确当前每个批次处于什么状态；
3. 说明共享回归页里哪些断言属于 `WBS-3.3.3`。

## 2. 分批原则

当前统一按以下顺序推进：

1. `batch_structure_selection`
   - 先修结构选择异常
2. `batch_target_tags`
   - 再修主体 / 范围标签一致性
3. `batch_effect_tags`
   - 最后修动作效果标签一致性

原因：

1. 结构异常不修，UI 选择语义和规划阶段就不稳定；
2. 主体 / 范围标签不修，后面的设计统计与检索口径会失真；
3. 效果标签清理应建立在前两层稳定之后再收口。
