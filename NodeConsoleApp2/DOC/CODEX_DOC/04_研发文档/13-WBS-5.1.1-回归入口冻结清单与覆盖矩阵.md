# WBS-5.1.1 回归入口冻结清单与覆盖矩阵

最后整理时间：2026-04-14

状态：`当前有效`

## 1. 文档用途

本文件固定“当前发布候选版必须保留哪些回归入口，以及它们分别覆盖什么风险”。

它不替代人工验收清单，也不替代专项 probe 页说明。

## 2. 入口分层

### 2.1 正式保留入口

1. `mock_ui_v11.html`
   - 覆盖：真实主流程、主界面说明、运行时主链烟测
2. `test/codex_regression_runner.html`
   - 覆盖：共享自动断言、WBS 范围过滤、回归基线
3. `DOC/CODEX_DOC/05_测试文档/02_验收清单/00-当前待验收功能总入口.md`
   - 覆盖：当前活跃人工验收顺序与待验收节点总览

### 2.2 辅助验证入口

1. `test/skill_contract_probe.html`
   - 覆盖：技能契约摘要、问题码与批次闭环
2. `test/battle_presentation_probe.html`
   - 覆盖：展示层驱动与动作反馈
3. `npm run test:acceptance-clicks`
   - 覆盖：主流程、展示层 probe、展示配置器的自动点击验收烟测
   - 前置：本地服务 `3101` 与 Chrome CDP `9222`
4. `test/skill_editor_io_test.html`
5. `test/buff_editor_io_test.html`
6. `test/level_editor_io_test.html`
7. `test/level_map_editor_io_test.html`

### 2.3 当前不进入正式冻结入口的页面

1. 历史 smoke / 旧验证页
2. 未挂到当前 WBS 节点的探索性页面
3. 清单仍不可理解的验收页

## 3. 覆盖矩阵

| 入口 | 主要覆盖面 | 不覆盖什么 |
| --- | --- | --- |
| `mock_ui_v11.html` | 主流程真实操作、页面自解释、主链烟测 | 批量契约统计、作者工具细节 |
| `test/codex_regression_runner.html` | 共享自动断言、WBS 范围过滤 | 人工体验与视觉判断 |
| `00-当前待验收功能总入口.md` | 当前待验收节点顺序、清单入口 | 自动断言本身 |
| `npm run test:acceptance-clicks` | 自动点击主流程、展示 probe、配置器 JSON/workspace 往返 | 用户逐条人工批注 |
| `skill_contract_probe.html` | 技能契约专项分析 | 主流程真实游玩 |
| `battle_presentation_probe.html` | 展示层专项演示 | 规则层数值正确性 |
| `*_io_test.html` | 作者工具输入输出链 | 主流程结算链 |

## 4. 冻结规则

1. 新增测试页不自动成为正式冻结入口。
2. 只有在 WBS 节点、验收清单和当前总入口中都有明确定位的页面，才可进入正式入口层。
3. 历史页面可以继续存在，但默认归为“辅助验证”或“历史归档”。
