# WBS-3.2.3.2 本地地图资源库替换设计

创建时间：2026-04-12

## 1. 背景

当前 `WBS-3.2.3.2` 地图编辑工作区虽然已经具备背景图、节点素材图和立绘图的资源引用链，但默认资源仍然是演示性质的 `svg` 资源，无法满足当前验收对“正式本地图片资源库”的要求。

用户已明确指定两组本地资源目录：

1. 地图背景图目录：
   - `source/map/`
2. 关卡节点 icon 目录：
   - `source/scene_icon/`

本轮目标不是做上传器，而是把这两组目录直接接成 `WBS-3.2.3.2` 的正式默认可选资源库。

## 2. 目标

本轮只做以下事情：

1. 地图背景图选择器默认改为消费 `source/map/*.jpeg`
2. 节点素材图选择器默认改为消费 `source/scene_icon/*.png`
3. 中央视图和节点主图层真实加载这些本地图片文件
4. 地图包导出结构仍保持短引用，不把绝对路径直接写进地图节点

## 3. 非目标

本轮不做：

1. 本地资源上传功能
2. 通用资源库管理器
3. 立绘资源体系重做
4. 地图包 schema 大改

## 4. 方案选择

采用方案 A：

1. 直接替换当前样例包中的 `assetLibrary.backgrounds`
2. 直接替换当前样例包中的 `assetLibrary.nodeArts`
3. 当前演示用 `svg` 背景图与节点图不再作为 `WBS-3.2.3.2` 默认可选项

不采用双库并存方案，原因是：

1. 验收时容易混入低质量演示资源
2. 页面可选项会变得含混
3. 当前阶段更需要“默认正式资源库”，不是“兼容旧 demo”

## 5. 数据口径

地图包导出结构保持不变：

1. `maps[].backgroundRef`
2. `maps[].nodes[].artRefs.nodeArt`

真实图片路径仍集中维护在：

1. `assetLibrary.backgrounds[].src`
2. `assetLibrary.nodeArts[].src`

这意味着：

1. 编辑页仍通过短引用切换资源
2. 页面渲染仍通过资源表查真实路径
3. 后续要接正式资源管理器时，不需要推翻当前短引用结构

## 6. 实现点

本轮直接修改：

1. `assets/data/level_map_pack_v1.example.json`
2. `assets/data/level_map_pack_v1.invalid.example.json`
3. `test/level_map_editor_page.test.mjs`
4. `test/level_map_pack_io.test.mjs`

当前预期保持不改或只做最小兼容：

1. `script/editor/level/LevelMapAssetResolver.js`
2. `script/editor/level/LevelMapEditorPage.js`
3. `script/editor/level/LevelMapPackIO.js`

## 7. 验收结果

本轮完成后，应满足：

1. `背景图` 下拉框中出现的是 `source/map/` 下的正式背景图
2. `节点素材图` 下拉框中出现的是 `source/scene_icon/` 下的正式节点图
3. 切换后，中央视图背景和节点图层都加载真实本地图片
4. 现有短引用导出链仍然可用
