## 1. 文档职责

它的职责只有三件事：

1. 固定首轮技能契约异常的统计口径；
2. 把异常从“口头印象”收敛成“问题码 + 技能样本”；
3. 为后续逐项修复提供稳定输入。

3. 选择数量与选择模式的结构越界。

## 4. 代表性异常簇

### 4.1 选择结构越界

代表样本：

1. `skill_block`
2. `skill_bone_repair`
3. `skill_escape`
4. `skill_aegis`

典型问题：

1. `selection_count_exceeds_candidates`
2. `single_selection_count_invalid`

问题含义：

1. 技能声明了单选，但 `selectCount` 不是 `1`；
2. 技能声明的可选部位数量小于 `selectCount`；
3. 这类问题会直接污染 UI 选择语义、规划阶段和说明文案。

### 4.2 主体 / 范围标签漂移

代表样本：

1. `skill_regroup`
2. `skill_hold_the_line`
3. `skill_smash`
4. `skill_shockwave_copy_1770041956468`

典型问题：

1. `tag_scope_mismatch`
2. `tag_subject_mismatch`

问题含义：

1. `tags` 里的 `SUBJECT_* / SCOPE_*` 与真实 `target.subject / target.scope` 不一致；
2. 设计文档、检索、平衡统计会因此出现两套口径。

### 4.3 直接动作效果标签漂移

代表样本：

1. `skill_shockwave`
2. `skill_savage_charge`
3. `skill_skull_cracker`
4. `skill_artery_slice`

典型问题：

1. `unexpected_effect_tag`
2. `missing_effect_tag`

问题含义：

1. `tags` 写了某个效果，但 `actions / buffRefs` 无法直接推导；
2. 或者运行时动作已经体现该效果，但 `tags` 没同步；
3. 这类问题会让“技能说明写了什么”和“运行时实际做了什么”脱钩。

### 4.4 alias 继承型异常

代表样本：

1. `skill_bone_repair -> skill_block`
2. `skill_smash -> skill_hold_the_line`
3. `skill_escape -> skill_shockwave_copy_1770041956468`

问题含义：

1. alias 技能不会天然消失原技能的契约问题；
2. 如果 canonical skill 本身存在结构或标签冲突，alias 会把冲突完整继承到敌人侧；
3. 所以后续修复应优先修 canonical skill，再复核 alias 投影。

## 5. 当前优先修复顺序

建议按下面顺序承接后续修复：

1. 先修 `selection_count_exceeds_candidates` 与 `single_selection_count_invalid`
   - 这是最直接的结构错误
2. 再修 `tag_scope_mismatch / tag_subject_mismatch`
   - 统一说明口径与检索口径
3. 最后修 `unexpected_effect_tag / missing_effect_tag`
   - 收口设计标签与运行时动作事实

原因：

1. 结构越界不修，后面的标签和文案都不稳定；
2. 主体 / 范围标签不修，矩阵统计本身就会失真；
3. 动作效果标签漂移应建立在前两层稳定之后再统一清理。

## 6. 使用方式

后续任何技能一致性修复，都建议遵守：

1. 先在 `skill_contract_probe.html` 复现问题码；
2. 再修改技能配置或标签；
3. 修改后重新跑 `codex_regression_runner.html`；
4. 若某一批问题已收口，再更新本归档或新开下一轮归档，不覆盖原始统计基线。
