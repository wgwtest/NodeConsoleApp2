# 当前阶段验收说明

## 1. 当前阶段完成了什么

当前文档已更新到“引擎主流程改造第二阶段”，重点仍然不在 UI，而在战斗运行时闭环。

已完成：

- 技能执行层从旧字段迁移到新 schema
  - 不再依赖 `skillConfig.type`
  - 不再依赖 `skillConfig.value`
  - 不再依赖 `skillConfig.targetType`
  - 改为按 `skills[].actions[].effect` 执行
- 技能 `buffRefs.apply/remove` 已接入主流程
- Buff wrapped 数据已接入运行时
  - `buffs_v2_7.json` 现在会正确取 `buffs` 字典
- Buff 运行时兼容层已补
  - 支持大写 action 名称
  - 支持 `payload.value/valueType`
  - 支持数组形式 `statModifiers`
- 受击后置事件已补入主伤害链路
  - `BATTLE_TAKE_DAMAGE`
  - `BATTLE_DEFEND_POST`
- `ATTACK` 类 Buff 已接入真实反击桥接
  - `BuffSystem` 不再只是占位
  - `CoreEngine` 已接住 `BUFF_ATTACK_REQUEST`
  - `buff_counter` 已可在实战中打出反击伤害
- 敌人行为从“固定第一个技能”升级为规则规划
  - 引入 `EnemyActionPlanner`
- 敌人缺失技能已加运行时别名映射
  - 目的是先保证战斗闭环，不是最终数据方案
- 行动前事件已接入
  - `BATTLE_ACTION_PRE` 现在能真正取消行动
- AP 生命周期补齐
  - 玩家回合 AP 重置
  - 敌人回合 AP 重置
  - 执行后才扣 AP
  - 被取消的行动不扣 AP

## 2. 本阶段已经做过的验证


结果：

- 当前没有静态语法错误

### 2.2 关键路径核对

已经确认：

- `CoreEngine` 中已不存在旧执行入口依赖：
  - `skillConfig.type`
  - `skillConfig.value`
  - `skillConfig.targetType`
- `BATTLE_ACTION_PRE` 已进入技能执行前置阶段
- `BATTLE_TAKE_DAMAGE` 已进入伤害落地后阶段
- `BATTLE_DEFEND_POST` 已进入防御结算后阶段
- 敌人 AP 初始化、回合重置、技能消耗已经接入
- `buffs_v2_7.json` 的 wrapped 结构已被运行时正确拆包
- 敌人当前引用的缺失技能已有运行时别名可解析

### 2.3 浏览器实战烟测

已在浏览器无头自动化环境中完成 `level_1_1` 最小实战验证。

对照结果：

- 不挂 `buff_counter`
  - 回合结束后敌人 HP 仍为 `50`
  - 日志中不存在反击记录
- 挂 `buff_counter`
  - 回合结束后敌人 HP 变为 `40`
  - 日志出现：`Hero triggered 反击 and dealt 10 HP to 哥布林战士.`

这说明：

- 反击不是日志假象
- `ATTACK` Buff 已经真实影响战斗结果
- 新增桥接已经进入主流程
