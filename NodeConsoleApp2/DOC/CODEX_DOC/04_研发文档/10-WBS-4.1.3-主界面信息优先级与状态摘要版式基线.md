# WBS-4.1.3 主界面信息优先级与状态摘要版式基线

记录时间：2026-04-11 10:50:31 +0800

状态：`已实现 / 已自测 / 待人工验收`

对应 WBS：`#89 / WBS-4.1.3 主界面信息优先级与状态摘要版式基线`

## 1. 本节点解决的问题

在 `WBS-4.1.1 / WBS-4.1.2` 完成后，主流程按钮语义和模态开关规则已经统一，但主界面信息层次仍有三个问题：

1. 战斗页缺少“当前目标 / 当前状态 / 下一步操作”的统一摘要层。
2. 系统菜单、关卡选择页已经有摘要信息，但它们的结构和战斗页不是同一套语义。
3. 玩家看到的信息仍偏研发视角，真正需要先看的操作提示不够靠前。

因此 `WBS-4.1.3` 的职责不是新增功能，而是把现有流程的关键信息重新排成稳定层级。

## 2. 本轮实现

### 2.1 主界面摘要条

更新：

1. `mock_ui_v11.html`
2. `script/ui/UI_AttentionGuide.js`
3. `mock_ui_v11.css`

新增能力：

1. 在主战斗页加入 `#battleStateSummary` 挂载点。
2. `UI_AttentionGuide` 在原有顶部 badge 之外，新增三卡摘要条：
   - `当前目标`
   - `战斗态势`
   - `当前操作`
3. 摘要条只消费现有 engine 状态：
   - `battlePhase`
   - `timeline.phase`
   - `currentTurn`
   - `playerSkillQueue`
   - `playerData.stats`
4. 摘要条不写回规则层，不影响战斗、Buff、敌人行为或存档。

### 2.2 模态摘要统一语义类

更新：

1. `script/ui/UI_SystemModal.js`
2. `mock_ui_v11.css`

本轮把主菜单与关卡选择页的摘要区统一挂到同一套语义类上：

1. `summary-section`
2. `summary-metric-grid`
3. `summary-metric-card`
4. `summary-detail-card`

对应摘要类型：

1. `growth`
2. `content-sources`
3. `story-progress`
4. `prebattle-build`

这意味着后续如果再做新手说明、错误反馈或正式 UI 皮肤替换，可以继续沿用这套摘要骨架，而不是重新拼一套 DOM。

### 2.3 共享回归页补 scope

更新：

1. `test/codex_regression_runner.html`
2. `test/d1_information_priority.test.mjs`

新增：

1. `WBS-4.1.3` 的 scope 过滤
2. 主界面摘要条存在与状态切换断言
3. 系统菜单 / 关卡选择摘要语义类断言
4. `mock_ui_v11.html` 挂载点断言

## 3. 当前边界

本节点边界固定为“只改展示层信息优先级”：

1. 不改技能规则。
2. 不改 Buff 计算。
3. 不改敌人行为。
4. 不改存档结构。
5. 不引入新的主流程按钮。

允许的变化只有：

1. 摘要信息更早出现。
2. 摘要文案更贴近玩家操作顺序。
3. 摘要 DOM 与样式语义统一。
