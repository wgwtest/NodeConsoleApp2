## 1. 验收目标

确认战斗中的 runtime snapshot 只在内存里推进，只有点击 `保存` 后才会把最小必要字段写入持久化存档。

本次修订重点：

1. 去掉巨量 `runtime` 原始 dump，改成可直接肉眼核对的短对象。
2. `保存` 后必须出现显式反馈，并更新槽位描述。
3. `save_game` 中只要求检查最小必要字段，不要求人工筛整段 JSON。

## 2. 验收入口

1. `http://127.0.0.1:3101/mock_ui_v11.html`
2. 浏览器控制台

## 3. 操作步骤

1. 打开 `http://127.0.0.1:3101/mock_ui_v11.html`
2. 点击 `开始冒险`
3. 点击 `关卡选择`
4. 选择 `幽暗森林边缘`
5. 在战斗主界面点击 `重锤`
6. 点击任意一个敌方 `head` 槽位
7. 点击 `提交规划`
8. 点击右上角齿轮按钮
9. 点击 `存档 / 读档`
10. 先观察 `存档位 1` 当前描述
11. 点击 `存档位 1` 的 `保存`
12. 观察页面内是否立即出现 `已保存到存档位 1`
13. 在浏览器控制台执行：

```js
(() => {
  const runtime = JSON.parse(localStorage.getItem('save_game')).runtime;
  return {
    scene: runtime.currentScene,
    turn: runtime.turn,
    phase: runtime.phase,
    timelinePhase: runtime.battleState?.timelinePhase,
    playerQueue: runtime.queues?.player?.length,
    enemyQueue: runtime.queues?.enemy?.length
  };
})()
```

## 4. 通过标准

应同时满足：

1. 页面立即出现 `已保存到存档位 1`
2. `存档位 1` 的描述已刷新，不再是 `空 | 关卡: -` 或 `主菜单`
3. `存档位 1` 的描述里至少能看到：
   - `幽暗森林边缘`
   - `回合: 1`
4. 控制台短对象返回结果满足：
   - `scene = "BATTLE_LOOP"`
   - `turn = 1`
   - `phase = "PLANNING"`
   - `timelinePhase = "READY"`
   - `playerQueue >= 1`
   - `enemyQueue >= 1`

## 5. 失败表现

任一情况均判未通过：

1. 点 `保存` 后页面无任何即时反馈
2. 槽位描述没有更新，仍然显示 `空` 或 `主菜单`
3. 控制台短对象缺少 `scene / turn / phase / timelinePhase / queues`
4. `scene` 不是 `BATTLE_LOOP`
5. `turn` 不是 `1`
