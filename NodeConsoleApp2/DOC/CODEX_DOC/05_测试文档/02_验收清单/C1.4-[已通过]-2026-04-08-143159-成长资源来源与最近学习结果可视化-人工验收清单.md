# C1.4 成长资源来源与最近学习结果可视化人工验收清单

对应 WBS：`#74 / C1.4 成长资源来源与最近学习结果可视化`【通过】

## 1. 本清单在验什么

只验证 `C1.4`：

1. 主菜单除了已有 KP/已学技能外，还会显示最近成长来源。
2. 主菜单会显示最近学习结果。
3. 技能树提交后这些信息会即时刷新。

## 2. 验收入口

1. 主入口：`http://127.0.0.1:3101/mock_ui_v11.html`
2. 自动回归辅助入口：`http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 使用 `http://127.0.0.1:3101/...` 打开页面。
2. 建议从一个可正常进入主菜单的存档或新局开始。
3. 若主菜单已经残留旧状态，先刷新页面再执行。

## 4. 为什么这些步骤能证明 C1.4

1. `C1.4` 验证的不是“技能树能不能打开”，而是“成长变化能不能在主菜单被解释清楚”。
2. 所以必须同时观察：
   - 最近成长来源
   - 最近学习结果
   - 提交后是否即时刷新

## 5. 人工验收步骤

### 5.1 读取最近成长来源

步骤：

1. 打开 `mock_ui_v11.html`
2. 在欢迎页点击 `新游戏`，或点击 `读取存档` 进入已有主流程

观察点：

1. 主菜单有 `成长摘要`
2. 摘要中存在 `最近成长来源`

### 5.2 学习并回看最近学习结果

步骤：

1. 在主菜单点击 `技能树 / 构筑`
2. 任意学习至少 1 个技能
3. 点击 `提交并关闭`

观察点：

1. 回到主菜单后仍在 `成长摘要`
2. 摘要中存在 `最近学习结果`
3. 能看到最近学习技能名或明确学习结果说明
4. 能看到 `消耗 X KP` 与 `剩余 X KP`

### 5.3 即时刷新验证

步骤：

1. 记录提交前主菜单的 KP 或学习结果
2. 重复一次技能学习并 `提交并关闭`

观察点：

1. 主菜单无需刷新即可变化
2. `最近学习结果` 内容同步更新
3. `最近成长来源` 与 `关卡选择` 入口未被破坏

## 6. 通过标准

以下全部满足才算通过：

1. 主菜单出现 `最近成长来源`
2. 主菜单出现 `最近学习结果`
3. 技能树提交后主菜单即时刷新
4. 新信息不会破坏原有 `关卡选择` 与 `技能树 / 构筑` 路径

## 7. 自动回归辅助核对

回归页应看到以下 3 项为 `PASS`：

1. `DataManager 会保留最近学习结果进度字段`
2. `UI_SkillTreeModal 提交后会记录最近学习结果`
3. `UI_SystemModal 会展示最近成长来源与学习结果`
