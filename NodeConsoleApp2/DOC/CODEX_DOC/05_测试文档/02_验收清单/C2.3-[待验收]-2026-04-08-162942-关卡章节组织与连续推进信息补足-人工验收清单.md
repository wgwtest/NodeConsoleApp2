# C2.3 关卡章节组织与连续推进信息补足人工验收清单

对应 WBS：`#78 / C2.3 关卡章节组织与连续推进信息补足`

## 1. 本清单在验什么

只验证 `C2.3`：

1. 关卡选择页是否新增了可理解的 `章节推进总览`。
2. 每张故事关卡卡片是否说明了 `章节节点` 和 `推进关系`。
3. 当前推荐与后续解锁方向是否会随故事进度变化。

## 2. 验收入口

1. 主入口：`http://127.0.0.1:3101/mock_ui_v11.html`
2. 自动回归辅助入口：`http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 使用 `http://127.0.0.1:3101/...` 打开页面。
2. 建议优先使用新档或低进度存档，以便更容易看见 `当前推荐` 和 `后续节点`。
3. 如果使用已有存档，不要求数字必须和示例一致，但要求章节总览与实际进度相符。

## 4. 为什么这些步骤能证明 C2.3

1. `C2.3` 不是新增关卡内容，而是让玩家看懂故事推进结构。
2. 所以必须同时看：
   - 关卡选择页顶部总览
   - 单张故事关卡卡片的节点关系
   - 通关后再次进入关卡选择时，推荐目标是否推进

## 5. 人工验收步骤

### 5.1 顶部章节总览

步骤：

1. 打开 `mock_ui_v11.html`
2. 点击 `开始冒险`
3. 点击 `关卡选择`

观察点：

1. 页面顶部出现 `章节推进总览`
2. 总览区能看见：
   - 当前章节名
   - 已完成数量
   - 已解锁数量
   - `当前推荐`
   - `后续解锁`
3. 这些信息是面向玩家的说明，不是内部 ID 或调试字段

### 5.2 单关卡节点关系

继续观察关卡卡片：

1. 每张 story 关卡卡片都有 `章节节点`
2. 每张 story 关卡卡片都有 `推进关系`
3. 已完成节点、当前推荐节点、后续未解锁节点在文案上能区分
4. 对未解锁节点，能看懂“完成哪一关后解锁”

### 5.3 连续推进更新

步骤：

1. 选择当前推荐的故事关卡并完成一局
2. 回到 `关卡选择`

观察点：

1. `章节推进总览` 中的已完成数量发生推进
2. `当前推荐` 会切换到下一故事节点，或明确显示章节已完成
3. 新解锁的后续节点不再继续显示为纯锁定态

## 6. 通过标准

以下全部满足才算通过：

1. 玩家在进入关卡前能直接看懂当前章节和当前推进位置
2. 玩家能从卡片上看懂每个节点的顺序与解锁关系
3. 完成当前推荐关卡后，章节总览与推荐目标会真实更新，而不是静态文案
4. 新增总览与提示不会破坏原有关卡选择、已完成态、未解锁态

## 7. 自动回归辅助核对

回归页应看到以下 3 项为 `PASS`：

1. `故事关卡提供连续推进路径`
2. `DataManager 构建故事关卡章节总览与当前推荐`
3. `UI_SystemModal 会渲染章节推进总览与节点关系`
