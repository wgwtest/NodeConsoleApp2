## 4. 人工验收步骤

### 4.1 I/O 页观察默认 pack 映射

1. 打开 `http://127.0.0.1:3101/test/level_editor_io_test.html`
2. 点击 `Load from ../assets/data/levels.json`
3. 观察点：
   - 页面显示 `Loaded Levels (8)`
   - 列表中 `level_1_1` 的名称为 `幽暗森林边缘`
   - `level_1_1` 的敌人池显示为 `pool_story_goblin_edge`

### 4.2 runtime probe 观察映射结果

1. 打开 `http://127.0.0.1:3101/test/level_runtime_probe.html`
2. 观察点：
   - `Pack Source` 显示为 `file`
   - `Schema Version` 显示为 `levels_v1_wrapped`
   - `Resolved Level Name` 显示为 `幽暗森林边缘`
   - `Resolved Enemy Members` 显示为 `goblin_01@1`
3. 点击 `Instantiate Probe Level`
4. 观察点：
   - 页面状态显示 `Instantiated level_1_1.`
   - `Instantiated Enemy Names` 显示为 `哥布林战士`

### 4.3 回归补充佐证

1. 打开 `http://127.0.0.1:3101/test/codex_regression_runner.html`
2. 确认以下条目为 `PASS`
   - `关卡 pack schema 与敌人池配置映射对齐`
   - `验收关卡与样本敌人/技能已配置`
3. 确认既有回归没有新增失败

## 5. 通过标准

满足以下全部条件才算通过：

1. I/O 页能够直接展示 `level_1_1 -> pool_story_goblin_edge`
2. runtime probe 页能够把默认文件 pack 解析为 `goblin_01@1`
3. 点击实例化后，运行时能解析出 `哥布林战士`
4. 回归页中关卡 pack 映射相关条目保持 `PASS`
