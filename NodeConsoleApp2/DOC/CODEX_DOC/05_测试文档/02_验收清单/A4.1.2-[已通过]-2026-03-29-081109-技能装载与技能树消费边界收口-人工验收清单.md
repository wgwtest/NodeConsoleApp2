## 4. 人工验收步骤

### 4.1 主流程验证

1. 打开 [mock_ui_v11.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/mock_ui_v11.html)
2. 按标准主流程执行：
   - 点击 `开始冒险`
   - 点击 `关卡选择`
   - 选择 `幽暗森林边缘`
   - 点击 `打开技能树`
   - 选择一个未学习且可学习的技能，例如 `修理护甲`
   - 点击 `提交并关闭`
3. 观察点：
   - 技能树模态窗可以正常打开
   - 提交后模态窗关闭
   - 主界面技能区新增刚学习的技能
   - 没有出现新的装载错误或脚本错误
4. 在同一页面继续执行：
   - 选择任一技能，例如 `重锤`
   - 部署到任一合法技能槽，例如敌方 `胸部`
   - 点击 `提交规划`
   - 点击 `执行`
5. 观察点：
   - 页面能进入下一回合
   - 学习后的技能在下一回合前后仍可见
   - 规划提交和执行均可完成

### 4.2 保存路径补充验证

1. 再次点击 `打开技能树`
2. 观察点：
   - 刚才已学习的技能不应再呈现为“未学习”
   - 技能树与主界面技能区状态一致
3. 关闭模态窗

### 4.3 回归补充佐证

1. 打开 [codex_regression_runner.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/test/codex_regression_runner.html)
2. 确认以下条目为 `PASS`
   - `DataManager 暴露技能目录边界 API`
   - `UI_SkillTreeModal 仅消费技能目录边界`
   - `CoreEngine.learnSkill 仅消费 DataManager 技能边界`
3. 确认既有回归没有新增失败

## 5. 通过标准

满足以下全部条件才算通过：

1. `mock_ui_v11.html` 中技能树可正常打开、学习、关闭
2. 学习后的技能会真实进入主界面技能区，而不是只在模态窗中变化
3. 学习后的技能在继续执行一回合后仍然存在
4. 主流程未因本轮改动回退，仍可正常提交规划并进入下一回合
5. 回归页中与技能边界相关的补充条目保持 `PASS`
