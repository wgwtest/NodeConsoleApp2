## 5. 人工验收步骤

### 5.1 正式编辑器入口与默认状态

1. 打开 `http://127.0.0.1:3101/test/level_editor_v1.html`
2. 观察点：
   - 页面标题为 `正式关卡编辑器 v1`
   - 顶部明确写出“正式关卡编辑器主入口”
   - `状态栏` 显示已从 `../assets/data/levels.json` 加载关卡包
   - 左侧关卡列表先显示 `story` 关卡，再显示 `acceptance` 关卡
   - 默认选中 `level_1_1`
   - `结构校验` 区域显示 `未发现结构问题`

### 5.2 关卡字段、连接与敌人池编辑

1. 在 `level_editor_v1.html` 中保持 `level_1_1` 选中
2. 修改以下字段：
   - `名称` 改成 `幽暗森林边缘·人工验收样本`
   - `KP` 改成 `7`
   - 在 `nextLevelIds` 复选框里取消勾选 `level_1_2_story`
3. 点击 `保存当前关卡`
4. 在 `波次与敌人池` 区域修改：
   - `enemyPoolName` 改成 `幽暗森林边缘敌人池·人工验收样本`
   - `敌人池成员` 改成两行：

```text
goblin_story_headhunter@1
enemy_acceptance_guard@2
```

5. 点击 `保存波次与敌人池`
6. 点击 `导出 JSON`
7. 观察点：
   - 导出区 JSON 中 `levels.level_1_1.name = 幽暗森林边缘·人工验收样本`
   - `levels.level_1_1.rewards.kp = 7`
   - `levels.level_1_1.flow.nextLevelIds = []`
   - `enemyPools.pool_story_goblin_edge.name = 幽暗森林边缘敌人池·人工验收样本`
   - `enemyPools.pool_story_goblin_edge.members` 为两项：
     - `goblin_story_headhunter@1`
     - `enemy_acceptance_guard@2`

### 5.3 Runtime Override 与独立运行时消费

1. 在 `level_editor_v1.html` 点击 `应用 Runtime Override`
2. 观察点：
   - 页面下方状态显示 `Runtime override active`
3. 打开或刷新 `http://127.0.0.1:3101/test/level_runtime_probe.html`
4. 观察点：
   - `Pack Source = override`
   - `Resolved Level Name = 幽暗森林边缘·人工验收样本`
   - `Resolved Enemy Members` 同时包含：
     - `goblin_story_headhunter@1`
     - `enemy_acceptance_guard@2`
5. 点击 `Instantiate Probe Level`
6. 观察点：
   - 页面状态显示 `Instantiated level_1_1.`
   - `Instantiated Enemy Names` 显示两名敌人，不再是 `-`

### 5.4 清理与回退验证

1. 返回 `level_editor_v1.html`
2. 点击 `清除 Runtime Override`
3. 再次回到 `level_runtime_probe.html`
4. 点击 `Reload Runtime`
5. 观察点：
   - `Pack Source` 恢复为 `file`
   - `Resolved Level Name` 恢复为默认值 `幽暗森林边缘`

### 5.5 自动回归辅助核对

1. 打开 `http://127.0.0.1:3101/test/codex_regression_runner.html`
2. 确认以下条目为 `PASS`
   - `DataManager 构建关卡内容来源分层摘要`
   - `level_editor_v1 提供正式关卡编辑入口`
   - `level_runtime_probe 提供运行时消费入口`
3. 确认本轮没有新增与关卡编辑器直接相关的失败项

## 6. 通过标准

满足以下全部条件才算通过：

1. 正式编辑器页面能明确解释自己的用途，并以 `level_editor_v1.html` 作为主入口
2. 页面内可以直接修改关卡名、奖励、连接关系、敌人池名称和敌人池成员
3. 导出的 JSON 与页面修改结果一致
4. 写入 override 后，独立运行时页能读到并实例化这份关卡产物
5. 清除 override 后，运行时页能回退到默认文件来源
