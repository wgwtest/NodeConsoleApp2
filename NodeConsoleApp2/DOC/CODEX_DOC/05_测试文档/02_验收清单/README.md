## 3. 当前不再放在这里的清单

以下类型不再留在当前有效区：

1. 没有明确 WBS 节点来源的清单
2. 使用临时标题、阶段标题、泛化标题的清单
3. 被新策略或新清单替代的旧清单
4. 跨多个 WBS 节点的组合性过程清单

## 4. 使用约束

从现在开始：

1. 新增人工验收清单前，先确认对应 WBS 节点已存在
2. 如果没有 WBS 节点，先补 issue / WBS，再补清单
3. 当前有效清单应优先使用页面入口、点击步骤、观察点和通过标准
4. 不再把“读代码确认”写成当前有效人工验收项
5. 当前有效清单文件名统一采用 `WBS编号-[状态]-日期-时间-内容-人工验收清单.md`
6. 如果未来发生目录迁移或文件改名，必须同步修正对应 GitHub Issue / Project 中的路径引用
