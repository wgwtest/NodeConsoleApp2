# C2.4 story / acceptance / 编辑器样本层级口径清理人工验收清单

对应 WBS：`#79 / C2.4 story / acceptance / 编辑器样本层级口径清理`

## 1. 本清单在验什么

只验证 `C2.4`：

1. 主菜单是否能解释 `story / 验收样本 / 作者样本工具页` 的分工。
2. `验收样本` 页面是否仍然只承载验收样本，不混入作者工具页语义。
3. 作者工具页是否明确说明自己不属于故事推进或验收样本入口。

## 2. 验收入口

1. 主入口：`http://127.0.0.1:3101/mock_ui_v11.html`
2. 作者样本工具页：
   - `http://127.0.0.1:3101/test/level_editor_io_test.html`
   - `http://127.0.0.1:3101/test/level_runtime_probe.html`
3. 自动回归辅助入口：`http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 使用 `http://127.0.0.1:3101/...` 打开页面。
2. 本节点主要验“说明口径与入口边界”，不要求完成战斗。
3. 如需观察作者样本 override 链路，可选做，但不是本节点唯一通过条件。

## 4. 为什么这些步骤能证明 C2.4

1. `C2.4` 的目标不是新增内容，而是澄清内容来源边界。
2. 所以必须同时看：
   - 运行时主菜单里的入口说明
   - 验收样本页里的用途说明
   - 作者工具页自己的用途说明

## 5. 人工验收步骤

### 5.1 主菜单入口分工

步骤：

1. 打开 `mock_ui_v11.html`
2. 在欢迎页点击 `新游戏`，或点击 `读取存档` 进入已有主流程【没有这个按钮】

观察点：

1. 主菜单存在 `内容入口说明`【没有，我再强调一次，你这个验收清单可能已经过期了，最新状态已经将你的状态冲过去了】
2. 说明区明确提到：
   - `故事关卡`
   - `验收样本`
   - `作者样本工具页`
3. 说明区明确写出 `作者样本工具页` 不在游戏菜单
4. 主菜单按钮仍然只有：
   - `关卡选择`
   - `验收样本`
   - 不会出现把作者样本当成运行时按钮的情况

### 5.2 验收样本页边界

步骤：

1. 在主菜单点击 `验收样本`

观察点：

1. 页面标题仍是 `选择验收样本`
2. 文案仍然强调“不影响故事推进”
3. 页面可以提示作者样本链路在哪，但不会把作者样本页当成验收卡片
4. 卡片列表里仍然只有验收样本关卡

### 5.3 作者样本工具页边界

步骤：

1. 分别打开：
   - `level_editor_io_test.html`
   - `level_runtime_probe.html`

观察点：

1. 两页都明确写出 `作者样本工具页`
2. 两页都明确写出：
   - 不属于故事推进关卡
   - 不属于验收样本入口
3. 页面仍然清楚解释自己的真实职责：
   - I/O 页负责注入 override
   - runtime probe 页负责重新加载并消费 override

## 6. 通过标准

以下全部满足才算通过：

1. story、验收样本、作者样本三类入口在命名和说明上可区分
2. 主菜单没有把作者样本错误做成运行时入口
3. 验收样本页没有被作者样本语义污染
4. 作者工具页能独立说明自己的职责与边界

## 7. 自动回归辅助核对

回归页应看到以下 4 项为 `PASS`：

1. `level_editor_io_test 提供 runtime bridge 验证入口`
2. `level_runtime_probe 提供运行时消费入口`
3. `DataManager 构建关卡内容来源分层摘要`
4. `UI_SystemModal 会解释 story/验收样本/作者样本入口分工`
