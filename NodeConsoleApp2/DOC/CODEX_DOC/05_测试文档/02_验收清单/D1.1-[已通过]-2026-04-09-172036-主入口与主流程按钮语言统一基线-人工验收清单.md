# D1.1 主入口与主流程按钮语言统一基线人工验收清单

【通过】

对应 WBS：`#87 / D1.1 主入口与主流程按钮语言统一基线`

## 1. 本清单在验什么

只验证 `D1.1`：

1. 主流程关键按钮的语言是否统一
2. 玩家是否能区分 `提交规划` 与 `执行回合`
3. 时间线控制按钮是否带有明确上下文
4. 文案收口是否未破坏原有主流程

## 2. 验收入口

1. 主流程入口：`http://127.0.0.1:3101/mock_ui_v11.html`
2. 自动回归辅助入口：`http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 使用 `http://127.0.0.1:3101/...` 打开页面。
2. 如果页面已残留旧战斗状态，先刷新页面。
3. 本清单只验按钮语言，不验 `D1.2 / D1.3` 的模态规则或版式层级。

## 4. 为什么这些步骤能证明 D1.1

1. `D1.1` 验的不是战斗规则，而是玩家在主流程第一眼看到的动作语义。
2. 因此要同时看：
   - 时间线按钮是否还保留模糊的 `开始`
   - 技能树入口是否与主菜单统一
   - 回合主执行按钮是否已经从模糊的 `执行` 收口成 `执行回合`
3. 如果这些都成立，并且主流程仍能正常推进，就说明 D1.1 的目标已经成立。

## 5. 人工验收步骤

### 5.1 时间线按钮语义

步骤：

1. 打开 `mock_ui_v11.html`
2. 观察页面顶部时间线控制区

观察点：

1. 不再显示孤立的 `开始`
2. 当前按钮显示为 `时间轴开始`

### 5.2 技能树入口语义

步骤：

1. 进入主流程
2. 观察技能列表下方入口按钮

观察点：

1. 按钮显示为 `技能树 / 构筑`
2. 不再显示旧文案 `打开技能树`

### 5.3 提交与执行语义分离

步骤：

1. 任意拖入一个技能到技能槽
2. 先点击 `提交规划`
3. 再观察右侧回合控制按钮

观察点：

1. 技能面板仍使用 `提交规划`
2. 右侧主执行按钮显示为 `执行回合`
3. 两个按钮的动作语义不再混淆

### 5.4 主流程非阻断烟测

步骤：

1. 走一轮主流程：
   - `开始冒险`
   - `关卡选择`
   - 进入关卡
   - `技能树 / 构筑` 可正常打开
   - `提交规划`
   - `执行回合`

观察点：

1. 文案调整后主流程仍可推进
2. 没有因为按钮改名导致事件失效

## 6. 通过标准

以下全部满足才算通过：

1. 时间线按钮使用 `时间轴开始`
2. 技能树入口统一为 `技能树 / 构筑`
3. 主执行按钮明确为 `执行回合`
4. 主流程不被按钮改名破坏
