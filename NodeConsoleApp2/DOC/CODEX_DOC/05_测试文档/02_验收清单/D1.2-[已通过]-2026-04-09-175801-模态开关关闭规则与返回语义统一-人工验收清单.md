# D1.2 模态开关关闭规则与返回语义统一人工验收清单

对应 WBS：`#88 / D1.2 模态开关、关闭规则与返回语义统一`【通过】

## 1. 本清单在验什么

只验证 `D1.2`：

1. 嵌套页面的返回按钮是否明确写出返回目标。
2. 战斗内打开系统菜单时，页面是否明确解释右上角关闭按钮会回战斗。
3. 技能树页面是否直接解释：
   - `提交并关闭`
   - `直接关闭`
   - `返回上一页`

## 2. 验收入口

1. 主入口：`http://127.0.0.1:3101/mock_ui_v11.html`
2. 自动回归辅助入口：`http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 使用 `http://127.0.0.1:3101/...` 打开页面。
2. 如果页面残留旧状态，先刷新。
3. 本清单只验模态规则语义，不验 `D1.3` 的版式优先级。

## 4. 为什么这些步骤能证明 D1.2

1. `D1.2` 验的不是新功能，而是“玩家是否在页面内读得懂模态层规则”。
2. 因此要同时看：
   - 关卡选择 / 读档页的返回按钮是否写明返回目标
   - 战斗内系统菜单是否明确解释 `×` 会回战斗
   - 技能树是否明确解释 `提交并关闭` 与 `直接关闭` 的差别
3. 如果这些都成立，就说明 D1.2 已经把“退出语义”和“返回语义”从隐含规则改成了页面内规则。

## 5. 人工验收步骤

### 5.1 关卡选择页的返回语义

步骤：

1. 打开 `mock_ui_v11.html`
2. 点击 `开始冒险`
3. 进入 `游戏菜单`
4. 点击 `关卡选择`

观察点：

1. 页面内有说明：点击关卡卡片会直接进入对应关卡
2. 页脚按钮显示为 `返回游戏菜单`
3. 不再只是泛化的 `返回`

### 5.2 读档页的返回语义

步骤：

1. 刷新页面
2. 在欢迎页点击 `读取存档`

观察点：

1. 页面内有说明：读取后会直接进入存档对应的页面
2. 页脚按钮显示为 `返回欢迎页`【显示的不是“返回欢迎页”，是“返回游戏菜单，请确认”】
3. 不再只是泛化的 `返回`

### 5.3 战斗内系统菜单的关闭语义

步骤：

1. 从欢迎页进入主流程：
   - `新游戏` 或 `读取存档`
   - `关卡选择`
   - 选择任一关卡
2. 进入战斗主界面后点击 `设置`

观察点：

1. 菜单说明块中明确写出：右上角关闭按钮会直接返回战斗
2. 菜单按钮列表仍然用于进入其他页面
3. 玩家不需要猜测 `×` 是回菜单还是回战斗

### 5.4 技能树的提交与直接关闭语义

步骤：

1. 在战斗主界面点击 `技能树 / 构筑`

观察点：

1. 技能树页面内直接写出：
   - `提交并关闭会保存本次学习并返回上一页`
   - `直接关闭不提交未保存更改，只返回上一页`
2. 不需要依赖聊天解释才能理解两个动作差别

### 5.5 主流程非阻断烟测

步骤：

1. 在技能树中任选一项技能学习并点击 `提交并关闭`
2. 返回战斗后继续：
   - 拖技能到槽位
   - `提交规划`
   - `执行回合`

观察点：

1. D1.2 的语义改动没有破坏主流程
2. 技能树、系统菜单、关卡选择仍然可正常工作

## 6. 通过标准

以下全部满足才算通过：

1. 关卡选择页脚明确为 `返回游戏菜单`
2. 读档页脚明确为 `返回欢迎页`
3. 战斗内系统菜单明确解释 `×` 会回战斗
4. 技能树明确解释 `提交并关闭` 与 `直接关闭`
5. 主流程未被 D1.2 改坏
