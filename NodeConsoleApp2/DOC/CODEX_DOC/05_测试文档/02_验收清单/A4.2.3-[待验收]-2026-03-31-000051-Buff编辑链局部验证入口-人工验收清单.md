## 4. 人工验收步骤

### 4.1 产物注入验证

1. 在 [buff_editor_io_test.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/test/buff_editor_io_test.html) 点击 `Load from ../assets/data/buffs_v2_7.json`
2. 点击 `Inject Runtime Probe Sample`
3. 观察点：
   - Buff 列表中 `buff_counter` 的名称变为 `反击·运行时覆写样本`
4. 点击 `Apply Runtime Override`
5. 观察点：
   - 页面状态栏显示 override 已生效

### 4.2 运行时消费验证

1. 打开或刷新 [buff_runtime_probe.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/test/buff_runtime_probe.html)
2. 观察点：
   - `Pack Source` 显示为 `override`
   - `Resolved Name` 显示为 `反击·运行时覆写样本`
3. 点击 `Apply Probe Buff`
4. 观察点：
   - 页面状态显示 `Applied buff_counter: 反击·运行时覆写样本`
   - `Active Buffs` 列表出现 `buff_counter | 反击·运行时覆写样本`

### 4.3 恢复验证

1. 返回 [buff_editor_io_test.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/test/buff_editor_io_test.html)
2. 点击 `Clear Runtime Override`
3. 观察点：
   - 页面状态栏显示 override 已清除
4. 再次刷新 [buff_runtime_probe.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/test/buff_runtime_probe.html)
5. 观察点：
   - `Pack Source` 恢复为 `file`
   - `Resolved Name` 恢复为默认名称 `反击`

### 4.4 回归补充佐证

1. 打开 [codex_regression_runner.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/test/codex_regression_runner.html)
2. 确认以下条目为 `PASS`
   - `DataManager 支持 Buff pack 运行时覆写桥`
   - `buff_editor_io_test 提供 runtime bridge 验证入口`
   - `buff_runtime_probe 提供运行时消费入口`
3. 确认既有回归没有新增失败
