## 1. 验收目标

确认“继续游戏”严格受显式保存控制：

1. 不保存时，刷新后不能继续当前战斗。
2. 保存后，刷新后可以继续当前战斗。
3. 恢复后的战斗可以继续执行并推进到下一回合。

说明：

1. 本节点不再要求“执行后自动把新回合继续写回 `save_game`”。
2. 现在的正确契约是：战斗内快照实时更新内存，只有点击 `保存` 才落盘。

## 2. 验收入口

1. `http://127.0.0.1:3101/mock_ui_v11.html`

## 3. 操作步骤

### 3.1 未保存分支

1. 在浏览器控制台执行：

```js
localStorage.removeItem('save_game');
localStorage.removeItem('save_game_slot_1');
localStorage.removeItem('save_game_slot_2');
localStorage.removeItem('save_game_slot_3');
location.reload();
```

2. 点击 `开始冒险`
3. 点击 `关卡选择`
4. 选择 `幽暗森林边缘`
5. 不点击 `保存`，直接刷新页面
6. 再次点击 `开始冒险`
7. 观察主菜单按钮

### 3.2 显式保存分支

1. 回到 `A3.1`，按其步骤完成一次显式保存
2. 刷新 `mock_ui_v11.html`
3. 再次点击 `开始冒险`
4. 观察主菜单按钮中是否出现 `继续游戏`
5. 点击 `继续游戏`
6. 观察是否恢复到刚才保存的战斗界面
7. 点击 `执行`
8. 等待自动执行结束
9. 观察回合数是否进入 `2`

## 4. 通过标准

应同时满足：

1. 在 `3.1 未保存分支` 中，主菜单里不出现 `继续游戏`
2. 在 `3.2 显式保存分支` 中，主菜单里出现 `继续游戏`
3. 点击 `继续游戏` 后，页面回到战斗主界面，而不是回到选关页
4. 恢复后的战斗点击 `执行` 后能够正常跑完本回合
5. 执行结束后，回合数进入 `2`
6. 整个过程无白屏、无明显报错、无卡死

## 5. 失败表现

任一情况均判未通过：

1. 未保存时仍然出现 `继续游戏`
2. 显式保存后仍然没有 `继续游戏`
3. 点击 `继续游戏` 后没有回到战斗界面
4. 恢复后点击 `执行` 无法推进
5. 执行结束后回合数没有进入 `2`
