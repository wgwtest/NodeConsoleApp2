# WBS-3.3.3 技能异常修复分批闭环与状态回写人工验收清单

对应 WBS：`#98 / WBS-3.3.3 技能异常修复分批闭环与状态回写`

## 1. 本清单在验什么

只验证 `WBS-3.3.3`：

1. 首批结构异常是否已经正式收口；
2. 页面是否能直接解释“当前节点负责什么、后续批次还有什么”；
3. 共享回归页是否能通过 `scope=WBS-3.3.3` 聚焦当前节点；
4. 分批与状态回写文档是否已经形成正式入口。

## 3. 为什么这些步骤能证明 WBS-3.3.3

1. `WBS-3.3.3` 验的不是“所有技能问题都已经修完”，而是“异常是否进入了正式修复闭环，并且首批结构问题已经真的被消掉”；
2. 因此需要同时看：
   - 批量校核页是否解释了分批闭环；
   - 分批文档是否明确写了当前状态；
   - 共享回归页是否能只看当前节点范围；
   - 首批结构问题是否已经从运行时异常里消失。

## 4. 人工验收步骤

### 4.1 分批闭环面板检查

步骤：

1. 打开 `skill_contract_probe.html`
2. 找到 `WBS-3.3.3 异常分批闭环` 面板

观察点：

1. 页面明确写出 `WBS-3.3.3`
2. 页面明确写出：
   - `当前节点负责的批次`
   - `已关闭批次`
   - `后续批次`
   - `非本节点`
3. `首批结构选择异常` 显示为 `已关闭批次`

### 4.2 首批结构异常收口检查

步骤：

1. 在同一页观察日志区或异常分批面板

观察点：

1. `selection_count_exceeds_candidates` 不再作为未关闭问题出现于首批结构批次
2. `single_selection_count_invalid` 不再作为未关闭问题出现于首批结构批次

### 4.3 共享回归范围检查

步骤：

1. 打开 `codex_regression_runner.html?scope=WBS-3.3.3`

观察点：

1. 页面会明确解释自己是 `共享回归页`
2. 页面会明确解释：
   - `当前节点范围`
   - `scope=`
   - `非本节点失败项`
   - `只运行当前范围`
3. 当前范围下，至少以下断言为 `PASS`
   - `DataManager 会按修复批次汇总技能异常`
   - `skill_contract_probe 会解释异常分批闭环`
   - `codex_regression_runner 提供 scope 范围说明`

### 4.4 文档状态回写检查

步骤：

1. 打开 `08-WBS-3.3.3-技能异常修复分批与状态回写.md`

观察点：

1. 文档明确写出 `batch_structure_selection / batch_target_tags / batch_effect_tags`
2. 文档明确写出首批结构批次已关闭
3. 文档明确写出后续两个批次的剩余 issue 数量

## 5. 通过标准

以下全部满足才算通过：

1. 批量校核页已能解释 `WBS-3.3.3` 的分批闭环；
2. 首批结构异常已被正式收口；
3. 共享回归页已支持当前节点范围聚焦；
4. 分批与状态回写文档已形成正式入口。
