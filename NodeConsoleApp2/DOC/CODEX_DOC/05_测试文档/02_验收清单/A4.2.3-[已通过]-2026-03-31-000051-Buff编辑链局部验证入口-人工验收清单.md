## 1. 验收目标

确认当前项目已经具备一条明确、可复现的“Buff 配置产物 -> 运行时消费”验证路径。

这 3 个页面的分工如下：

1. `buff_editor_io_test.html`：把运行时覆写样本写入本地 override 存储
2. `buff_runtime_probe.html`：在一个独立页面里重新加载运行时并消费这个 override
3. `codex_regression_runner.html`：给这条链路补自动回归佐证

## 2. 验收入口

1. `http://127.0.0.1:3101/test/buff_editor_io_test.html`
2. `http://127.0.0.1:3101/test/buff_runtime_probe.html`
3. `http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 所有页面均通过本地静态服务打开
2. 验收开始前，先在 `buff_editor_io_test.html` 点击 `Clear Runtime Override`
3. `buff_editor_io_test.html` 只负责写入 override，不负责证明运行时消费成功
4. `buff_runtime_probe.html` 会独立重新装载运行时，用来证明 override 被真正消费
5. `codex_regression_runner.html` 只提供自动回归佐证，不替代人工理解页面职责

## 4. 人工验收步骤

### 4.1 产物注入验证

1. 打开 `http://127.0.0.1:3101/test/buff_editor_io_test.html`
2. 点击 `Load from ../assets/data/buffs_v2_7.json`
3. 点击 `Inject Runtime Probe Sample`
4. 观察点：
   - Buff 列表中 `buff_counter` 的名称变为 `反击·运行时覆写样本`
5. 点击 `Apply Runtime Override`
6. 观察点：
   - 页面状态栏显示 `Runtime override saved`
   - 页面下方 `Runtime Bridge` 区域显示 `Runtime override active`

### 4.2 运行时消费验证

1. 打开或刷新 `http://127.0.0.1:3101/test/buff_runtime_probe.html`
2. 观察点：
   - `Pack Source` 显示为 `override`
   - `Resolved Name` 显示为 `反击·运行时覆写样本`
3. 点击 `Apply Probe Buff`
4. 观察点：
   - 页面状态显示 `Applied buff_counter: 反击·运行时覆写样本`
   - `Active Buffs` 列表出现 `buff_counter | 反击·运行时覆写样本`

### 4.3 恢复验证

1. 返回 `http://127.0.0.1:3101/test/buff_editor_io_test.html`
2. 点击 `Clear Runtime Override`
3. 观察点：
   - 页面状态栏显示 `Runtime override cleared.`
4. 再次刷新 `http://127.0.0.1:3101/test/buff_runtime_probe.html`
5. 观察点：
   - `Pack Source` 恢复为 `file`
   - `Resolved Name` 恢复为默认名称 `反击`

### 4.4 回归补充佐证

1. 打开 `http://127.0.0.1:3101/test/codex_regression_runner.html`
2. 确认以下条目为 `PASS`
   - `DataManager 支持 Buff pack 运行时覆写桥`
   - `buff_editor_io_test 提供 runtime bridge 验证入口`
   - `buff_runtime_probe 提供运行时消费入口`
3. 确认既有回归没有新增失败

## 5. 通过标准

满足以下全部条件才算通过：

1. I/O 页面中可以通过按钮把 probe 样本注入到运行时桥
2. runtime probe 页面中可直接观察到 Buff 名称被覆写并被运行时消费
3. 清除 override 后，runtime probe 页面中的 Buff 名称恢复默认
4. 整个验证过程可重复执行，且不会永久污染默认数据
5. 回归页相关补充条目保持 `PASS`
