## 1. 验收目标

确认当前项目已经具备一条可重复的故事关卡推进链：

1. 关卡选择页会显示故事关卡的解锁状态。
2. 点击未解锁关卡不会误启动战斗。
3. 第一关首次通关后会解锁下一关。
4. 结算页中的 `再次挑战`、`前往下一关`、`返回主菜单` 三个去向可用。

## 2. 验收入口

1. `mock_ui_v11.html`
2. `test/codex_regression_runner.html`
3. 浏览器控制台

## 3. 前置条件

1. 使用本地静态服务打开页面。
2. 为避免旧存档干扰，验收前先在控制台执行：

```js
localStorage.removeItem('save_game');
location.reload();
```

## 4. 操作步骤

### 4.1 回归补充佐证

1. 打开 `http://127.0.0.1:3101/test/codex_regression_runner.html`
2. 确认以下条目为 `PASS`
   - `故事关卡提供连续推进路径`
   - `DataManager 构建故事关卡选择入口`
   - `DataManager 首次通关会解锁下一故事关卡`
   - `CoreEngine 结算态允许重新选择关卡`
   - `UI_SystemModal 会渲染结算去向按钮`
   - `UI_SystemModal 关卡选择遵守解锁状态`
3. 确认回归页没有新增失败项

### 4.2 主页面关卡选择验证

1. 打开 `http://127.0.0.1:3101/mock_ui_v11.html`
2. 点击 `开始冒险`
3. 在菜单中点击 `关卡选择`
4. 观察点：
   - 一共出现 `3` 张关卡卡片
   - 第一关为 `幽暗森林边缘`
   - 第二关为 `密林前哨`，带 `未解锁`
   - 第三关为 `废墟关隘`，带 `未解锁`
5. 点击 `密林前哨`
6. 观察点：
   - 页面仍停留在 `选择关卡`
   - 没有进入战斗主界面

### 4.3 主页面结算去向验证

1. 在 `选择关卡` 中点击 `幽暗森林边缘`
2. 等待页面进入战斗规划态
3. 在浏览器控制台执行：

```js
window.GameEngine.endBattle(true);
```

4. 观察点：
   - 模态窗标题为 `战斗胜利`
   - 模态窗出现以下按钮：
     - `再次挑战`
     - `前往下一关：密林前哨`
     - `返回主菜单`
5. 在控制台执行：

```js
JSON.parse(localStorage.getItem('save_game')).global.progress
```

6. 观察点：
   - `unlockedLevels` 包含 `level_1_2_story`
   - `completedLevels` 包含 `level_1_1`
   - `lastSettlement.nextLevelId === 'level_1_2_story'`

### 4.4 再次挑战验证

1. 在当前结算页点击 `再次挑战`
2. 观察点：
   - 重新进入战斗主界面
   - 控制台执行 `window.GameEngine.data.currentLevelData.id` 后结果为 `level_1_1`

### 4.5 返回主菜单验证

1. 在当前战斗中再次执行：

```js
window.GameEngine.endBattle(true);
```

2. 在结算页点击 `返回主菜单`
3. 观察点：
   - 模态窗标题切回 `游戏菜单`
   - 控制台执行 `window.GameEngine.fsm.currentState` 后结果为 `MAIN_MENU`

### 4.6 前往下一关验证

1. 在 `游戏菜单` 中再次点击 `关卡选择`
2. 选择 `幽暗森林边缘`
3. 进入战斗后，在控制台再次执行：

```js
window.GameEngine.endBattle(true);
```

4. 在结算页点击 `前往下一关：密林前哨`
5. 观察点：
   - 进入战斗主界面
   - 控制台执行 `window.GameEngine.data.currentLevelData.id` 后结果为 `level_1_2_story`
   - 控制台执行 `window.GameEngine.fsm.currentState` 后结果为 `BATTLE_LOOP`

## 5. 通过标准

满足以下全部条件才算通过：

1. 回归页中的 6 条 B2 相关条目保持 `PASS`
2. 关卡选择页正确展示 `3` 张故事关卡与解锁状态
3. 点击未解锁关卡不会启动战斗
4. 首次通关第一关后，存档中写入下一关解锁结果
5. `再次挑战`、`返回主菜单`、`前往下一关` 三条去向都能进入正确状态

## 6. 失败表现

任一情况均判未通过：

1. 故事关卡数量不对，或锁定状态显示错误
2. 点击未解锁关卡后直接进入战斗
3. 结算页缺少任一去向按钮
4. `save_game.global.progress.unlockedLevels` 未写入 `level_1_2_story`
5. `再次挑战`、`返回主菜单`、`前往下一关` 任一点击后未进入预期状态
