## 3. 前置条件

1. 所有页面都通过 `http://127.0.0.1:3101/...` 打开，不要直接双击本地文件
2. 开始前先在 `buff_editor_io_test.html` 点击一次 `Clear Runtime Override`
3. 名词解释与页面协作边界：
   - 这里说的 `Buff 数据`，就是 `buffs_v2_7.json` 这份 Buff 配置数据
   - `Apply Runtime Override` 不会重写 `buffs_v2_7.json` 文件；它只会把“当前页里这份 Buff 数据”另存为浏览器 `localStorage` 里的临时 JSON 数据
   - 这里说的“本地”是指当前浏览器对这个站点保存的数据，不是工程目录里的本地文件，也不是几个页面直接共享的一段内存
   - `buff_runtime_probe.html` 与刷新后重新进入运行时的 `mock_ui_v11.html`，会在重新加载时读这份临时 Buff 数据
   - 已经打开但没有刷新的页面不会自动变化
   - `buff_editor_v4.html` 每次直接读取默认文件 `buffs_v2_7.json`，不读取这份临时数据，因此不会被本页直接改写
4. 本节点之所以需要跨页，是为了验证“这一页把 Buff 数据写进浏览器临时存储后，另一页刷新后真的会读到并用上它”，而不是在单页内自循环证明
## 4. 人工验收步骤

### 4.1 默认 Buff 编辑页仍可正常工作

1. 打开 `http://127.0.0.1:3101/test/buff_editor_v4.html`
2. 观察点：
   - 页面标题为 `Buff Editor v4`
   - 顶部能看到 `Load buffs_v2_7.json`
   - 页面显示 `Loaded: 19 buffs (schema=buffs_v2_1_wrapped)`
   - 列表中能看到 `buff_counter / 反击`

### 4.2 默认运行时消费仍走文件 pack

1. 先打开 `http://127.0.0.1:3101/test/buff_editor_io_test.html`

4. 观察点：

   - `Resolved Name` 显示为 `反击`
5. 点击 `Apply Probe Buff`
6. 观察点：
   - 页面状态显示 `Applied buff_counter: 反击`
   - `Active Buffs` 列表出现 `buff_counter | 反击`
