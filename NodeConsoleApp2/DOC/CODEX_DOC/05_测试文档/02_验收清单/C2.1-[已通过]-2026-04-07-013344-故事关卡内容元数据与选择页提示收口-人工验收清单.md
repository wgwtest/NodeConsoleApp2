对应 WBS：【通过】

状态：`已通过`

## 1. 本清单是验证什么的

本清单不是验证“关卡能不能点进去”，而是验证：

1. 玩家在进入战斗前，能不能从关卡选择页直接看懂 3 个 story 关卡的差异
2. 新增提示信息是否破坏了原有的 `已完成 / 未解锁` 状态语义

如果这项通过，说明阶段 C 的内容扩充已经开始形成“可感知内容差异”，而不只是配置文件里多了几行数据。

## 2. 验收入口

主入口：

1. `mock_ui_v11.html`

辅助入口：

1. `test/codex_regression_runner.html`

## 3. 操作步骤

1. 打开 `mock_ui_v11.html`
2. 点击 `新游戏`
3. 在游戏菜单点击 `关卡选择`
4. 观察 3 个 story 关卡卡片

## 4. 必查项

### 4.1 三个 story 关卡都应出现内容提示

检查每张卡片是否都能看到：

1. `难度`
2. `敌人风格`
3. `奖励预览`
4. `构筑提示`

批注区：

【】

### 4.2 三个关卡提示内容应明显不同

检查重点：

1. 第一关应偏向 `弱点追击 / 快攻`
2. 第二关应偏向 `低血自疗 / 节奏拉扯`
3. 第三关应偏向 `残甲修补 / 重击压迫`

批注区：

【】

### 4.3 奖励预览应可直接识别

检查重点：

1. 卡片上能直接看见 `EXP`
2. 卡片上能直接看见 `GOLD`
3. 卡片上能直接看见 `KP`

批注区：

【】

### 4.4 既有状态语义不能被破坏

检查重点：

1. 已完成关卡仍应保留 `已完成`
2. 未解锁关卡仍应保留 `未解锁`
3. 未解锁关卡不应因为新增提示而变成可正常点击

批注区：

【】
