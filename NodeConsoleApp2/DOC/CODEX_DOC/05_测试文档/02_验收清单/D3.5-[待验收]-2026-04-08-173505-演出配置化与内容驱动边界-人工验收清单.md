# D3.5 演出配置化与内容驱动边界人工验收清单

对应 WBS：`#82 / D3.5 演出配置化与内容驱动边界`

## 1. 本清单在验什么

只验证 `D3.5`：

1. 展示层是否已经拥有统一的 `presentation` 配置入口。
2. 模板来源、降级语义和场景脉冲是否能被人工直接观察。
3. 未知模板是否会稳定降级，而不是报错或卡死。
4. 这些配置化改动是否仍然保持低耦合，不阻断真实主流程。

## 2. 验收入口

1. 主验证入口：`http://127.0.0.1:3101/test/battle_presentation_probe.html`
2. 主流程烟测入口：`http://127.0.0.1:3101/mock_ui_v11.html`
3. 自动回归辅助入口：`http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 使用 `http://127.0.0.1:3101/...` 打开页面。
2. 优先从 `battle_presentation_probe.html` 开始，因为 `D3.5` 验的是展示层配置边界，不是伤害公式。

## 4. 为什么这些步骤能证明 D3.5

1. `D3.5` 的重点不是“再多做几个动画”，而是证明展示层已经有统一的配置消费口。
2. 所以必须观察：
   - 配置说明是否写清
   - `最近模板`
   - `最近配置来源`
   - `最近降级语义`
   - `最近场景脉冲`
3. 这些读数会直接告诉你：当前触发的是哪种模板、来自哪类配置输入、有没有走降级。

## 5. 人工验收步骤

### 5.1 读取 probe 页说明

步骤：

1. 打开 `battle_presentation_probe.html`

观察点：

1. 页面明确写出它是 `展示层验证页`
2. 页面有单独的 `演出配置边界` 说明
3. 页面写清 `最近模板 / 最近配置来源 / 最近降级语义 / 最近场景脉冲` 这组读数是干什么的

### 5.2 验证结构化配置入口

步骤：

1. 依次点击：
   - `模板：近战`
   - `模板：防御/格挡`
   - `模板：治疗`
   - `模板：Buff 上身`

观察点：

1. 每次点击后，`最近模板` 都会更新成对应类别
2. `最近配置来源` 显示为 `entry.presentation.template` 或同级结构化来源
3. `最近场景脉冲` 会随模板变动，而不是固定不变
4. 画面仍能看到对应前冲 / 格挡 / 治疗 / 状态提示

### 5.3 验证未知模板降级

步骤：

1. 点击 `模板：未知类别（降级）`

观察点：

1. 页面没有报错或卡死
2. `最近模板` 显示为 `default`
3. `最近降级语义` 显示为 `default`
4. `最近场景脉冲` 仍有值，而不是完全失效

### 5.4 主流程非阻断烟测

步骤：

1. 打开 `mock_ui_v11.html`
2. 按主流程完成：
   - `开始冒险`
   - `关卡选择`
   - 进入关卡
   - `提交规划`
   - `执行`

观察点：

1. 主流程仍可正常推进
2. 没有因为演出配置化新增额外确认步骤

## 6. 通过标准

以下全部满足才算通过：

1. probe 页解释清楚 `D3.5` 验的是什么
2. 结构化模板按钮能驱动对应读数与表现更新
3. 未知模板会降级到 `default`
4. 降级语义和场景脉冲可被人工直接观察
5. 主流程不被 `D3.5` 阻断

## 7. 自动回归辅助核对

回归页应看到以下 3 项为 `PASS`：

1. `BattlePresentationController 会消费结构化演出配置并归一化`
2. `BattlePresentationController 会把未知模板降级到 default`
3. `battle_presentation_probe 会解释演出配置边界与降级读数`
