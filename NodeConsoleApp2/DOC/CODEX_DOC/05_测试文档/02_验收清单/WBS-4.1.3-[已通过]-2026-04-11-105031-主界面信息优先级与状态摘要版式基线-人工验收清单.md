## 1. 本清单在验什么

只验证 `WBS-4.1.3`：

1. 主战斗页是否把“当前目标 / 战斗态势 / 当前操作”收敛成稳定摘要区。
2. 摘要区是否已经移到主界面底部第三行，不再压缩时间线和主战斗显示区。
3. 摘要内容是否会跟主流程状态一起变化。
4. 系统菜单与关卡选择页的摘要是否已经进入统一语义骨架。
5. 本节点是否仍然只改展示层信息优先级，而没有改动规则链。

## 3. 为什么这些步骤能证明 WBS-4.1.3

1. `WBS-4.1.3` 验的不是“战斗规则对不对”，而是“玩家是否能先看到现在该做什么，而且主画面不被摘要条挤压”。
2. 所以需要同时看：
   - 战斗主界面的摘要区是否独立成底部第三行；
   - 摘要内容是否会随着 `开始冒险 -> 关卡选择 -> 战斗规划 -> 执行回合` 切换；
   - 游戏菜单与关卡选择页是否也使用同一类摘要语义；
   - 全程是否没有引入新的规则按钮或隐藏流程。

## 4. 人工验收步骤

### 4.1 初始页版式检查

步骤：

1. 打开 `mock_ui_v11.html`

观察点：

1. 主时间线与主战斗显示区保持原有高度，没有被摘要区压缩。
2. 摘要区位于主界面底部第三行，而不是场景区顶部。
3. 摘要区至少包含三张卡：
   - `当前目标`
   - `战斗态势`
   - `当前操作`
4. 初始态应说明“先从游戏菜单进入关卡”，而不是误导你以为已经进入战斗。

### 4.2 主流程状态切换检查

步骤：

1. 点击 `开始冒险`
2. 在游戏菜单点击 `关卡选择`
3. 选择唯一关卡
4. 进入主战斗页后，任选技能并部署到技能槽
5. 点击 `提交规划`
6. 点击 `执行回合`

观察点：

1. 进入战斗页后，底部摘要区仍然存在。
2. 规划阶段摘要会强调：
   - 当前目标是选技能 / 部署技能；
   - 当前操作是提交规划或继续部署。
3. 当规划满足执行条件后，摘要会明确提示“可以执行回合”。
4. 执行阶段摘要会切换为“时间轴执行中 / 等待自动结算”之类的说明。
5. 在整个流程里，摘要区只更新信息，不会挤占主场景与时间线的可视空间。

### 4.3 系统菜单摘要统一检查

步骤：

1. 在主流程任意时点点击 `设置`
2. 观察 `游戏菜单`
3. 再点击 `关卡选择`

观察点：

1. `游戏菜单` 中可见：
   - `成长摘要`
   - `内容入口说明`
2. `关卡选择` 中可见：
   - `章节推进总览`
   - `关前构筑摘要`
3. 这些摘要看上去属于同一类信息卡，不再是完全不同的临时块。

### 4.4 回归辅助页 scope 检查

步骤：

1. 打开 `codex_regression_runner.html?scope=WBS-4.1.3`

观察点：

1. 页面明确显示当前 scope 为 `WBS-4.1.3`
2. 通过项应只围绕本节点：
   - `UI_AttentionGuide 提供主界面摘要条`
   - `UI_SystemModal 主菜单与关卡选择页使用统一摘要语义类`
   - `mock_ui_v11 提供主界面摘要条挂载点`
   - `mock_ui_v11.html 不再把主界面摘要条插入场景区顶部`

## 5. 通过标准

以下全部满足才算通过：

1. 主战斗页的关键信息已经前置为稳定摘要区。
2. 摘要区位于底部第三行，不再压缩主战斗显示区。
3. 摘要区会随主流程状态切换而变化。
4. 游戏菜单与关卡选择页的摘要区已经进入统一语义骨架。
5. 全程没有新增规则依赖，也没有打断原有主流程点击链。
