# C1.6 学习结果驱动的下一局可见反馈闭环人工验收清单

对应 WBS：`#76 / C1.6 学习结果驱动的下一局可见反馈闭环`

## 1. 本清单在验什么

只验证 `C1.6`：

1. 学习技能后进入下一局，主战斗界面会出现 `本局新增技能` 提示。
2. 提示会明确说明新技能已经自动加入技能池，本局可直接规划使用。
3. 玩家不必重新打开技能树，也能在下一局早期界面知道这次学习改变了什么。

## 2. 验收入口

1. 主入口：`http://127.0.0.1:3101/mock_ui_v11.html`
2. 自动回归辅助入口：`http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 使用 `http://127.0.0.1:3101/...` 打开页面。
2. 从一个可正常进入主菜单的新局或存档开始。
3. 如果已有旧学习记录干扰观察，建议刷新页面并重新走一轮“结算 -> 学习 -> 下一局”。

## 4. 为什么这些步骤能证明 C1.6

1. `C1.6` 验证的不是“技能树能不能学习”，而是“学习结果是否真正进入下一局玩家可见体验”。
2. 所以必须同时观察：
   - 学习后进入下一局是否出现新提示
   - 提示是否明确写出具体技能名
   - 提示是否解释“已自动加入技能池”
   - 玩家是否无需重新打开技能树也能获取这条信息

## 5. 人工验收步骤

### 5.1 先制造最近学习结果

步骤：

1. 打开 `mock_ui_v11.html`
2. 点击 `开始冒险`
3. 在主菜单点击 `技能树 / 构筑`
4. 学习至少 1 个技能并点击 `提交并关闭`

观察点：

1. 技能树关闭后返回主菜单
2. 最近学习结果已成功提交

### 5.2 进入下一局观察早期反馈

步骤：

1. 在主菜单点击 `关卡选择`
2. 进入唯一故事关卡
3. 等主战斗界面完全出现，不需要提交规划，也不需要重新打开技能树

观察点：

1. 技能操作区域顶部出现 `本局新增技能`
2. 提示里能看到刚学习的技能名
3. 提示里明确说明：
   - `已自动加入技能池`
   - 或同等含义文案

### 5.3 验证它确实是“下一局”反馈

观察点：

1. 这条提示出现在正式战斗主界面的早期区域
2. 玩家不重新打开技能树，也能理解“这次学习影响了本局可用技能”

## 6. 通过标准

以下全部满足才算通过：

1. 学习技能后进入下一局，战斗主界面出现 `本局新增技能`
2. 提示中包含具体技能名
3. 提示明确解释新技能已自动加入技能池
4. 不需要重新打开技能树，也能完成上述观察

## 7. 自动回归辅助核对

回归页应看到以下 3 项为 `PASS`：

1. `UI_SkillPanel 会在学习后的下一局显示新增技能反馈`
2. `UI_SkillPanel 不会在新结算之后继续显示旧学习反馈`
3. `DataManager 结算会记录 settledAt 时间戳`
