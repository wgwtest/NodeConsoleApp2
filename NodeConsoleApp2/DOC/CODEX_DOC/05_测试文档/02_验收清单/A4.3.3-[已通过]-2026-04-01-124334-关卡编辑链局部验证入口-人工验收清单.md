1. `level_editor_io_test.html`：把 probe 关卡样本写入本地 override 存储
2. `level_runtime_probe.html`：在一个独立页面里重新加载运行时并消费这个 override
3. `codex_regression_runner.html`：给这条链路补自动回归佐证

## 2. 验收入口

1. `http://127.0.0.1:3101/test/level_editor_io_test.html`
2. `http://127.0.0.1:3101/test/level_runtime_probe.html`
3. `http://127.0.0.1:3101/test/codex_regression_runner.html`

## 3. 前置条件

1. 所有页面均通过本地静态服务打开
2. 验收开始前，先在 `level_editor_io_test.html` 点击 `Clear Runtime Override`
3. `level_editor_io_test.html` 只负责写入 override，不负责证明运行时实例化成功
4. `level_runtime_probe.html` 会独立重新装载运行时并调用 `instantiateLevel`
5. `codex_regression_runner.html` 只提供自动回归佐证，不替代人工理解页面职责

## 4. 人工验收步骤

### 4.1 产物注入验证

1. 打开 `http://127.0.0.1:3101/test/level_editor_io_test.html`
2. 点击 `Load from ../assets/data/levels.json`
3. 点击 `Inject Runtime Probe Sample`
4. 观察点：
   - `level_1_1` 的名称变为 `幽暗森林边缘·运行时覆写样本`
   - `level_1_1` 的首个敌人池显示为 `pool_runtime_probe`
5. 点击 `Apply Runtime Override`
6. 观察点：
   - 页面状态栏显示 `Runtime override saved`
   - 页面下方 `Runtime Bridge` 区域显示 `Runtime override active`

### 4.2 运行时消费验证

1. 打开或刷新 `http://127.0.0.1:3101/test/level_runtime_probe.html`
2. 观察点：
   - `Pack Source` 显示为 `override`
   - `Resolved Level Name` 显示为 `幽暗森林边缘·运行时覆写样本`
   - `Resolved Enemy Members` 显示为 `enemy_acceptance_repair@1`
3. 点击 `Instantiate Probe Level`
4. 观察点：
   - 页面状态显示 `Instantiated level_1_1.`
   - `Instantiated Enemy Names` 显示为 `验收修理守卫`

### 4.3 恢复验证

1. 返回 `http://127.0.0.1:3101/test/level_editor_io_test.html`
2. 点击 `Clear Runtime Override`
3. 观察点：
   - 页面状态栏显示 `Runtime override cleared.`
4. 再次刷新 `http://127.0.0.1:3101/test/level_runtime_probe.html`
5. 观察点：
   - `Pack Source` 恢复为 `file`
   - `Resolved Level Name` 恢复为默认名称 `幽暗森林边缘`

### 4.4 回归补充佐证

1. 打开 `http://127.0.0.1:3101/test/codex_regression_runner.html`
2. 确认以下条目为 `PASS`
   - `DataManager 支持关卡 pack 运行时覆写桥`
   - `level_editor_io_test 提供 runtime bridge 验证入口`
   - `level_runtime_probe 提供运行时消费入口`
3. 确认既有回归没有新增失败

## 5. 通过标准

满足以下全部条件才算通过：

1. I/O 页面中可以通过按钮把 probe 关卡样本注入到运行时桥
2. runtime probe 页面中可直接观察到关卡名称与敌人池成员被覆写并被运行时消费
3. 清除 override 后，runtime probe 页面中的关卡名称恢复默认
4. 整个验证过程可重复执行，且不会永久污染默认数据
5. 回归页相关补充条目保持 `PASS`
