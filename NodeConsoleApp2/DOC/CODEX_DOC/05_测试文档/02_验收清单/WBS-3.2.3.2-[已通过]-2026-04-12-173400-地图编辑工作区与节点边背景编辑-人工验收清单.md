## 4. 人工验收步骤

### 4.1 页面用途与边界说明检查

步骤：

1. 打开 `test/level_map_editor_v1.html`

观察点：

1. 页头明确写出 `地图编辑工作区`
2. 页面明确写出 `JSON + image assets`
3. 页面明确出现 `背景图 / 节点素材图 / 立绘图` 等词
4. 页面明确写出：
   - `不会直接改战斗主流程`
   - `消费 levels.json 中已存在的 levelId`
   - `导入导出与 round-trip 校验不在本页完成`

### 4.2 背景图与图片资源检查

步骤：

1. 打开页面后，先确认左上工具栏里已经点过 `加载地图包样例`
2. 看左侧栏标题 `地图与结构`
3. 在左侧栏靠下找到 `地图设置` 区块
4. 在 `地图设置` 区块里找到 `背景图` 下拉框
5. 把 `背景图` 从当前值切到另一个值
   - 例如：从 `bg_map_glade_01 | 地图背景 01`
   - 切到 `bg_map_winter_02 | 地图背景 02 · 冬境`
6. 点击左上工具栏里的 `保存地图设置`

观察点：

1. `背景图` 下拉框正下方会出现 `背景图资源` 预览卡
2. 点击 `保存地图设置` 后，这张预览卡里的资源名称与图片会同步变化
3. 中间 `素材化地图舞台` 的整张背景会明显变化，而不是只换文字或渐变说明
4. 展开下方 `当前工作区导出预览` 后，当前地图的 `backgroundRef` 会同步变化
5. 这一组动作验证的是：
   - 左侧 `背景图` 下拉框负责选择 `backgroundRef`
   - `保存地图设置` 负责把这个值真正写回当前地图
   - 中央视图再根据 `backgroundRef -> assetLibrary.backgrounds[].src` 重新渲染真实背景图

### 4.3 节点素材图与立绘图检查

步骤：

1. 看右侧栏标题 `节点与边检查器`
2. 如果当前不在 `节点检查器`，先点右侧页签 `节点检查器`
3. 在中央舞台点击任意一个节点，或在右侧 `节点列表` 中点一个节点
4. 在右侧 `节点字段` 区块里找到：
   - `节点素材图`
   - `立绘图`
5. 分别切换这两个下拉框到其他值
   - `节点素材图` 现在默认来自 `source/scene_icon/`
   - 可选值例如：
     - `node_icon_elemental`
     - `node_icon_elf_archer`
     - `node_icon_goblin_warrior`
     - `node_icon_owlbear`
6. 点击右侧工具栏里的 `保存节点`

观察点：

1. `节点字段` 区块底部会出现两张预览卡：
   - `节点素材图`
   - `立绘图`
2. 点击 `保存节点` 后，两张预览卡里的资源会同步变化
3. 中央视图中，对应节点的主图层会变化
4. 该节点右下角的小立绘徽章也会变化
5. 展开 `当前工作区导出预览` 后，`artRefs.nodeArt / artRefs.portrait` 会同步变化
6. 这一组动作验证的是：
   - 右侧节点检查器确实能改节点图片资源引用
   - 改动不是只停留在表单里，而是会影响中央舞台和导出结构

### 4.4 节点增删改与拖拽检查

步骤：

1. 在右侧 `节点检查器` 顶部工具栏点击 `新增节点`
2. 在 `节点字段` 中至少修改以下几项：
   - `绑定 levelId`
   - `标签`
   - `标题`
   - `类型`
3. 点击 `保存节点`
4. 回到中央舞台，按住这个新节点拖拽到新位置
5. 如需确认结构变化，再展开 `当前工作区导出预览`

观察点：

1. 新节点会进入节点列表和中央舞台
2. 拖拽后位置变化
3. 导出预览中 `x / y` 会同步变化

### 4.5 边关系编辑检查

步骤：

1. 点击 `新增边`
2. 修改起点、终点、类型和分支文本
3. 点击 `保存边`
4. 点击中央舞台边标签或 `路线总览` 项

观察点：

1. 边列表新增
2. 中间舞台出现新的边和边标签
3. 点击边后会切到 `边检查器`
4. 选中边与路线总览同步高亮

### 4.6 布局直观性检查

步骤：

1. 直接观察地图编辑页首屏
2. 不滚到页面底部，只切换 `节点检查器 / 边检查器`

观察点：

1. 中央视图仍然是主视图，不再是窄条
2. 首屏没有明显文本溢出
3. 右侧不是长到需要疯狂滚动才能改边
4. 导出预览保持折叠，不抢主舞台首屏
