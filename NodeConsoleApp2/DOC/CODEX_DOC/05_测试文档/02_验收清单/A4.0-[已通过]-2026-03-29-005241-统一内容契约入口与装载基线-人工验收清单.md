## 4. 验收步骤

### 4.1 基线验证

1. 打开 [config.json](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/assets/data/config.json)
2. 确认可见：
   - 顶层存在 `contentRegistry`
   - `contentRegistry.skills.byTree.melee_v4_5.path = skills_melee_v4_5.json`
   - `contentRegistry.buffs.schemaVersion = buffs_v2_1_wrapped`
3. 打开 [mock_ui_v11.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/mock_ui_v11.html)
4. 执行主流程：
   - `开始冒险`
   - `关卡选择`
   - 选择 `幽暗森林边缘`
   - 选择任一技能，例如 `重锤`
   - 部署到任一合法槽位，例如敌方 `胸部`
   - 点击 `提交规划`
   - 点击 `执行`
5. 观察点：
   - 页面能进入战斗
   - 规划可提交
   - 执行后能进入下一回合
   - 没有新的内容装载错误
6. 在同一页面点击 `打开技能树`
7. 观察点：
   - 技能树能打开
   - 能看到技能节点列表
   - 能看到 KP 区域
   - 点击 `×` 可以关闭

### 4.2 扰动验证

这一步的目的不是破坏数据，而是证明运行时真的依赖 `config.json` 的统一内容入口。

1. 备份 [config.json](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/assets/data/config.json) 当前内容
2. 在 [config.json](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/assets/data/config.json) 中，只做一个最小扰动：
   - 将 `contentRegistry.skills.byTree.melee_v4_5.path` 临时改成一个不存在的文件名
   - 例如：`skills_melee_v4_5_missing.json`
3. 保存文件后刷新 [mock_ui_v11.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/mock_ui_v11.html)
4. 观察点：
   - 页面应出现明确的装载失败现象
   - 或无法正常进入依赖技能装载的流程
   - 总之页面表现必须随着配置扰动发生变化
5. 恢复 [config.json](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/assets/data/config.json) 原值
6. 再次刷新 [mock_ui_v11.html](/home/wgw/CodexProject/NodeConsoleApp2/NodeConsoleApp2/mock_ui_v11.html)
7. 观察点：
   - 页面恢复正常
   - 主流程可再次进入
   - 技能树可再次打开

满足以下全部条件才算通过：
