## 发现的问题

### 固定组合回放会稳定浪费 AP

- 级别：high
- ID：ap_underuse_fixed_combo
- 证据：基础破甲 + 轻流血：9/9 轮未打满 AP（第1轮 3/5，第2轮 3/5，第3轮 3/5）；连续建窗循环：8/8 轮未打满 AP（第1轮 4/5，第2轮 4/5，第3轮 4/5）；建窗后读窗循环：2/7 轮未打满 AP（第1轮 4/5，第7轮 3/5）
- 后续处理：下一版测试不应只硬编码短 combo，应基于已学技能池在“同回合同技能不可重复”的约束下搜索或枚举可用组合。

### 浅割与锯齿斩的低阶定位重叠

- 级别：high
- ID：low_tier_bleed_overlap
- 证据：浅割是 1AP/7 护甲门控伤害/+1W，锯齿斩是 1AP/4 护甲门控伤害/+2W；二者都属于低阶建窗器，玩家学会后更像固定叠加按钮，而不是清晰分工的选择
- 后续处理：重做低阶剑系时，应保留一个主建窗器，并把另一个改成明确的补刀、维窗、控场、防反或 AP 填充角色。

### 测试记录必须反向驱动技能设计

- 证据：本轮回放暴露的问题不能只停留在 HTML 展示里；后续重做剑系前，应先读取本记录的 findings，并把 AP 利用率、技能区分度、自然输出循环作为验收项
- 后续处理：每次生成 Skill 测试记录时同步输出 findings.md 与 report.json，供下一轮设计读取。
