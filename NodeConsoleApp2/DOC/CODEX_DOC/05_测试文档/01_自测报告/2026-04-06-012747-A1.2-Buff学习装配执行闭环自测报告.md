## 2. 本轮实现结论

### 2.1 主流程 Buff 样本收口

本轮把主流程中实际可学、可装配的 Buff 样本收口为以下四条：

1. `skill_regroup / 受击备甲`
   - 学习前置：`skill_savage_charge`
   - 结果：对自身施加 `受击获甲`
2. `skill_fortify / 堡垒姿态`
   - 学习前置：`skill_regroup`
   - 结果：立即回甲，并施加 `护甲回复`
3. `skill_aegis / 绝对防御`
   - 学习前置：`skill_fortify`
   - 结果：对自身施加 `护甲免伤`
4. `skill_1771769351059 / 锯齿斩`
   - 学习前置：`skill_heavy_swing`
   - 结果：对敌方施加 `流血`

### 2.2 主流程可观察性补强

3. `UI_BattleRow` 现在会把运行时混合状态按 `buff / debuff` 分流到正确的状态栏。

### 4.1 学习链路

结果：

1. 本轮重新核算后，`A1.2` 的完整学习链从空白构筑计算共需 `21 KP`：
   - `skill_block = 1`
   - `skill_savage_charge = 2`
   - `skill_regroup = 2`
   - `skill_fortify = 5`
   - `skill_aegis = 8`
   - `skill_heavy_swing = 1`
   - `skill_1771769351059 = 2`
2. 干净默认档位下，默认玩家初始可用 `KP = 30`，且默认已学技能 `skill_heal / skill_heavy_swing / skill_savage_charge` 在 `重置所有技能` 时会返还 `4 KP`，所以重置后应有 `34 KP`。
3. 在这个“干净默认档位 + 重置所有技能”的基线上，连续学习 `受击备甲 / 堡垒姿态 / 绝对防御 / 锯齿斩` 成功。
4. 学习提交后，技能池中出现上述四个技能。
5. `HP免伤演示 / 护甲免伤演示 / 受击获甲演示` 这类 `DEMO` 技能不再出现在正式技能树中。
6. 当前报告已修正此前错误表述：
   - 不是 `30 -> 13`
   - 而是干净默认档位 `34 -> 13`
7. 如果沿用旧存档，`开始冒险` 会优先载入旧成长数据；此时即便点了 `重置所有技能`，总 `KP` 也可能只有 `13`，这不能作为 `A1.2` 功能失效的证据。

### 4.2 第 1 回合：受击备甲 + 锯齿斩

部署：

1. `受击备甲` 放入己方槽位 `self:global:0`
2. `锯齿斩` 放入敌方槽位 `enemy:head:0`

结果：

1. 第 2 回合开始时，玩家 `BUFF` 栏出现 `受击获甲`
2. 第 2 回合开始时，敌人 `DEBUFF` 栏出现 `流血`
3. 日志中出现：
   - `Player: 受击备甲 applied 受击获甲 ...`
   - `Player: 锯齿斩 applied 流血 ...`
   - `[Buff] buff_bleed dealt 5 damage ...`

### 4.3 第 2 回合：堡垒姿态

部署：

1. `堡垒姿态` 放入己方槽位 `self:global:0`

结果：

1. 日志中出现：
   - `Player: 堡垒姿态 restored 20 armor on Player1:head.`
   - `Player: 堡垒姿态 applied 护甲回复 ...`
2. 这证明“即时回甲 + 挂后续 Buff”的复合链路已进入真实主流程。

### 4.4 第 3 回合：绝对防御

部署：

1. `绝对防御` 放入己方槽位 `self:global:0`

结果：

1. 日志中出现：
   - `Player: 绝对防御 applied 护甲免伤 ...`
   - 随后敌人发生攻击日志
2. 这证明防御型 Buff 已进入同回合的真实战斗结算链。
