# 剑系流血窗口模型与候选技能包自测报告

创建时间：2026-05-29 00:22:30 +0800

状态：已自测，待人工审定候选技能包

## 1. 本轮目标

本轮目标是把剑系流血从“堆砌流血类技能”调整为可推演的窗口模型，并生成一份不覆盖正式运行时包的 authoring 候选技能包。

核心约束：

1. 流血只使用 `buff_bleed.remaining`，不使用 stacks 口径。
2. 流血窗口可被低倍率读取，也可被高倍率消耗。
3. 不给单个技能写引擎特判，新增能力必须是通用 Skill-Buff 契约。
4. 候选技能包只写入 authoring 目录，不发布到 `assets/data/skills_melee_v4_5.json`。

3. `CoreEngine._applySkillBuffRefs()`
   - 支持 `buffRefs.remove[].consumeRemaining`。

4. `CoreEngine._validateSkillRequirements()`
   - 支持 `requirements.targetBuff`。

## 3. 候选流血支线

```text
剑击
└─ 浅割
   └─ 锯齿斩
      ├─ 深切
      └─ 撕裂伤口
         ├─ 失血踉跄
         ├─ 血涌斩
         │  └─ 断脉一剑
         └─ 饮血斩
            └─ 猩红收割
```

关键技能：

| 技能 | KP / AP | 作用 |
| --- | ---: | --- |
| 浅割 | 1 / 1 | 6 点生命伤害，增加 1 回合流血 |
| 锯齿斩 | 2 / 1 | 5 点生命伤害，增加 2 回合流血 |
| 深切 | 3 / 2 | 10 点生命伤害，增加 3 回合流血 |
| 撕裂伤口 | 3 / 2 | 5 点护甲伤害，施加 2 回合撕裂伤口；目标行动前给自己增加 2 回合流血 |
| 失血踉跄 | 3 / 1 | 目标至少 3 回合流血时，造成 4 点生命伤害并施加 2 回合减速 |
| 血涌斩 | 5 / 2 | 5 + `3 * min(W, 4)` 伤害，不消耗流血 |
| 饮血斩 | 5 / 2 | `3 * min(W, 4)` 治疗，不消耗流血 |
| 断脉一剑 | 8 / 3 | 目标 HP 低于 35% 且至少 3 回合流血时，15 + `8 * min(W, 4)` 伤害，消耗最多 4 回合流血 |
| 猩红收割 | 8 / 4 | 目标至少 5 回合流血时，20 + `6 * min(W, 8)` 伤害并回复 `3 * min(W, 8)` HP，消耗最多 8 回合流血 |
