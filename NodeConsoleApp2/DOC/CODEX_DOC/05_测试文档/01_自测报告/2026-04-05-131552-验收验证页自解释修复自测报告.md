## 1. 本轮目标

本轮不新增业务功能，只修复“验收验证页本身不自解释”的问题，重点解决以下两个缺陷：

1. 打开验证页后，不清楚本页用途、按钮作用、以及为什么这样点击就能证明对应能力。
2. `codex_regression_runner.html` 虽然新增了说明面板，但运行后会被测试自身改写 `body`，导致说明面板消失。

## 2. 修复结论

### 2.1 页面说明已直接固化到验证页正文

以下页面均已包含可见说明区块：

4. I/O 页与 runtime probe 页的职责分工
5. runtime probe 页关键字段的判读方式

### 2.2 回归页已修复“说明被测试污染”的根因

1. `test/codex_regression_runner.html` 中原有部分 DOM 测试直接覆盖 `document.body.innerHTML`
2. 这会把 `#usageGuide`、`#summary`、`#log` 一起删掉
3. 现已改成使用 `data-regression-fixture` 隔离测试夹具
4. 每条测试执行前后自动清理夹具，不再污染回归页自身说明与结果区域

### 2.3 回归断言已改成面向可见文案

1. 不再要求跨 HTML 标签的完整连续字符串
2. 改为匹配稳定的用户可见短语，例如：
   - `打开后自动执行全部回归用例`
   - `会重新加载`
   - `DataManagerV2.js`
