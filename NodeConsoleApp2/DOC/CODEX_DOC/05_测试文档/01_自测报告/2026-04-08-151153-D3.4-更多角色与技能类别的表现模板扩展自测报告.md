对应 WBS：`#81 / D3.4 更多角色 / 技能类别的表现模板扩展`

## 1. 本轮目标

把展示层从“只有单一攻击样本”推进到“按模板类别复用同一驱动”：

1. 支持至少 4 类表现模板：
   - `melee`
   - `guard`
   - `heal`
   - `status`
2. 未知模板必须稳定降级到 `default`。
3. probe 页要明确解释每个模板按钮为什么能证明对应能力。
4. 模板层仍然只消费事件与只读状态，不改战斗规则。

## 2. 本轮实现

涉及文件：

1. `script/ui/presentation/BattleAnimationDriver.js`
2. `script/ui/presentation/BattlePresentationController.js`
3. `script/ui/presentation/FighterPresenter.js`
4. `test/battle_presentation_probe.html`
5. `test/codex_regression_runner.html`

本轮新增：

1. 模板别名解析：
   - `block / defend / defense / parry -> guard`
   - `buff / debuff / support -> status`
   - `recovery -> heal`
2. `FighterPresenter.playTemplate()` 统一分发 `guard / heal / status / default`。
3. 场景节奏按模板触发不同脉冲，并把最近模板写入 `battle-scene.dataset`。
4. probe 页新增模板按钮与 `最近模板` 读数。
5. 自动回归新增 2 条专项断言：
   - `BattlePresentationController 会按别名解析表现模板类别`
   - `BattlePresentationController 会把未知模板降级到 default`
