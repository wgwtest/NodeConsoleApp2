# 技能与 Buff 渐进式解耦第四轮自测报告

创建日期：2026-05-21

状态：`已自测 / 待人工验收`

## 1. 本轮目标

本轮把“正式技能运行时能力分类”固化为自动测试，避免后续只靠人工扫 JSON 判断技能是否合理。

分类原则：

1. 不要求所有技能一次性通过；
2. 已能由当前通用机制表达的技能归为稳定项；
3. 数据字段存在歧义的技能归为语义待确认；
4. 缺 Buff、空运行时技能单独列出；
5. 不把复杂项伪装成已覆盖。

## 3. 固化的测试断言

新增测试：

```text
正式技能运行时能力分类保持可审计，复杂项不伪装为已覆盖
```

断言内容：

1. `西海岸` 必须仍被识别为 `missing_buff:buff_shield`；
2. `愤怒` 必须仍被识别为 `empty_runtime`；
3. `剑砍 / 重锤 / 诸神黄昏` 必须仍被识别为 `scaling_mismatch:ABS`；
4. `提桶跑路` 必须仍被识别为 `scaling_mismatch:PCT_CURRENT`；
5. 当前不允许出现未归档的 unsupported effectType；
6. 当前稳定项数量固定为 26。

这个测试不是为了阻止后续改设计，而是为了让后续改设计时必须显式更新分类结果，避免静默漂移。

## 4. 后续人工判断点

### 4.1 scaling mismatch

这 4 个技能需要决定 `scaling` 字段到底是什么意思：

1. 如果希望按 `atk` 缩放，应把 `amountType` 改为 `SCALING`；
2. 如果希望固定数值，建议删除 `scaling` 字段，避免编辑器和策划阅读误判；
3. 如果希望“固定值 + 缩放值”，需要新增明确的通用数值模型。

### 4.2 西海岸

`西海岸` 当前引用缺失 `buff_shield`。建议先确认护盾语义：

1. 一次性 HP 免伤；
2. 一次性护甲免伤；
3. 固定吸收池；
4. 上身护甲转移到腿部。

第 4 种会涉及部位级状态与护甲迁移，复杂度明显更高，不建议直接塞进 CoreEngine。

### 4.3 愤怒

`愤怒` 当前无 actions、无 buffRefs。建议确认：

1. 删除；
2. 作为隐藏测试节点；
3. 改为明确 Buff；
4. 改为直接动作技能。

## 6. 结论

当前测试体系已经从 Buff 原子能力推进到 Skill 直接动作层，并建立了正式技能运行时分类。

后续更适合进入“设计决策”而不是继续盲目写测试：`scaling` 语义、`buff_shield` 语义、`愤怒` 去留，这三类问题需要人工明确后再实现。
