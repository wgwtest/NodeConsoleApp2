# 设计说明子系统化重构记录

记录时间：2026-05-21

状态：`已执行 / 待人工验收`

## 1. 背景

原 `DOC/CODEX_DOC/02_设计说明/` 已经是正式设计说明根，但目录内 26 份稳定设计文档按历史编号扁平排列。随着主游戏流程、地图编辑器、技能编辑器、Buff 编辑器等分系统继续扩展，扁平结构不利于后续维护、引用和评审。

本轮参考 `CodeFactoryV2` 的“总纲 + 子系统目录 + 稳定入口 + 补充稿归档”方式，对本项目设计说明目录做适配式重构。

## 2. 执行原则

1. 保留 `DOC/CODEX_DOC/` 作为唯一正式本地文档根。
2. 保留现有 26 份设计说明的正文和历史文件名，不在本轮重写设计内容。
3. 将设计说明从扁平文档池调整为 `00_总纲 + S1-S7 子系统目录`。
4. 更新正式入口、当前计划、测试和过程文档中的设计说明路径，避免迁移后引用失效。
5. 纯过程材料、验收意见、原型截图仍分别归属 `06_过程文档/`、`05_测试文档/`、`08_原型与附图/`。

## 6. 后续约束

1. 新增设计说明时必须先判断归属子系统。
2. 已形成正式原型或截图基线的 UI 设计，应在设计说明中引用 `08_原型与附图/`，不要把原型过程塞进设计正文。
3. 若后续需要进一步重写各子系统主设计，应按子系统单独立项，避免一次性重写所有历史设计文档。
4. 本轮没有处理 2026-05-20 技能树 UI 未提交代码和原型材料归属。
