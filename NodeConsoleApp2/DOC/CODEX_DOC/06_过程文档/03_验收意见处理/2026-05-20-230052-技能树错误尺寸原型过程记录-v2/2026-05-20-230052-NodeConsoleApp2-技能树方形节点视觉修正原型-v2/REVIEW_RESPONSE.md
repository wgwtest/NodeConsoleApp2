# v2 评审回应

## 用户批注

用户指出当前运行态技能树存在以下问题：

- 整体界面偏暗，总图状态效果不明显。
- 长方形节点破坏原编辑器布局，仍然出现重叠。
- 技能名太小，KP 和加号占位不合理。
- 黑背景上黑节点只靠左侧亮条表达状态，不直观。
- 已学习、未学习、可学习效果不明显。
- 图例像机械制图，状态应直接体现在节点上。

## 判断

批注成立。

根因不是单个颜色或字号，而是上一轮运行态节点形态与编辑器布局事实冲突：编辑器按 `72 x 72` 节点和 `100px` 网格人工排布，运行态改成长方形后，哪怕复用 `editorMeta.x/y`，也会扩大横向占用并制造视觉重叠。
