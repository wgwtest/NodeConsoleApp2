## 3. 原因判断

1. `WBS-4.1.3` 的问题不是摘要卡内容本身，而是挂载位置错误，导致信息优先级优化反过来伤害了主战斗观察区。
2. `WBS-3.3.3` 的问题是我们仍从“issue / batch 视角”组织 probe 页，而用户要的是“按技能状态理解当前进展”的人工验收入口。
3. `WBS-4.3.7` 的问题有两层：
   - 工作区结构没有围绕“边调边看”组织；
   - `scene-impact` 仍使用整屏位移做冲击反馈，和当前验收口径冲突。
