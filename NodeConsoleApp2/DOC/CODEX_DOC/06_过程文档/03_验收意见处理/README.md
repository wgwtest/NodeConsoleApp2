# 验收意见处理 README

最后整理时间：2026-04-03

状态：`当前有效`

## 1. 目录用途

本目录用于承接用户在人工验收清单中的批注、结论和修正意见的正式处理文档。

## 2. 文档应包含什么

每份验收意见处理文档建议至少包含：

1. 来源验收清单
2. 用户意见摘录
3. 问题归类
4. 原因分析
5. 影响范围
6. 处理建议
7. 待用户审定项

## 3. 当前工程的执行规则

1. 用户在验收清单中给出明确批注后，如需调整计划、验收口径或代码方向，应先在本目录形成处理文档
2. 在用户审定处理意见前，不直接执行具有方向性影响的改计划或改实现动作
3. 处理文档形成后，应与对应验收清单、会话交接文档互相回链
