# 2026-04-13 WBS-3.2.3.2 逻辑地图空间数据驱动改造交接

## 1. 本轮目标

把地图编辑器从“直接编辑 DOM 像素坐标”的早期实现，改造成：

1. 地图包保存逻辑地图真值
2. 编辑页按当前画布尺寸做投影显示
3. 抽屉开关、视口变化不会改写地图真值
4. 预览页与 IO 页共同消费同一套数据口径

## 2. 根因判断

上一版问题不在样式细节，而在数据层缺位：

1. `nodes[].x / y` 被直接当成当前画布像素
2. 边几何直接依赖当前按钮像素位置
3. 画布尺寸变化后，节点显示和导出值混在一起
4. 地图包样例、编辑页、预览页没有统一到同一坐标语义

## 3. 已完成改动

1. `script/editor/level/LevelMapWorkspace.js`
   - 新增 `space / display / position` 归一化
   - 输入兼容旧 `x / y`
   - 导出统一收敛到 `nodes[].position`
2. `script/editor/level/LevelMapEditorPage.js`
   - 新增逻辑坐标到画布像素的投影函数
   - 新增像素到逻辑坐标的反投影函数
   - 拖拽、表单保存、边渲染全部切到逻辑空间
3. `script/editor/level/LevelMapPreviewPage.js`
   - 预览页改为消费 `space / display / position`
   - 背景和节点资源继续走 `assetLibrary`
4. `script/editor/level/LevelMapAssetResolver.js`
   - 背景铺放开始消费 `display.backgroundFit`
5. `assets/data/level_map_pack_v1.example.json`
   - 地图样例新增 `space / display`
   - 节点改为 `position`
6. `assets/data/level_map_pack_v1.invalid.example.json`
   - 非法样例同步切到新结构
7. `assets/data/contracts/level_map_definition.contract.json`
   - 契约字段改为 `space / display / position.x / position.y`

## 4. 当前稳定口径

1. 地图真值
   - `maps[].space`
   - `maps[].display`
   - `nodes[].position`
   - `edges[].fromNodeId / toNodeId`
2. 显示层派生
   - 节点在画布中的 `left/top`
   - 边路径与边标签像素位置
3. 不可导出状态
   - 抽屉开关
   - 当前视口尺寸
   - DOM 实际像素宽高

## 6. 后续建议

1. 下一步可以把 `display.nodeScale / nodeAnchor / edgeAnchor / edgeLabelMode` 进一步做成真正可调的编辑字段，而不是先只消费默认值。
2. 如果继续推进 `3.2.3.2`，建议把地图背景、节点素材、边样式的编辑入口彻底挂到同一套数据模型上，避免再出现“界面变了但导出不变”或“导出变了但界面没跟上”的分裂。
