# WBS-3.4.4 自动试玩与平衡报告自测记录

日期：2026-05-25

分支：`codex/enemy-design-20260522`

## 1. 本轮目标

推进 `WBS-3.4.4 自动试玩与平衡报告`，建立可重复执行的三章 30 关自动试玩工具，并把结果输出为机器可读 JSON 和人工可读 Markdown 摘要。

本轮只建立报告闭环：

1. 不调整敌人数值。
2. 不调整敌人技能。
3. 不调整关卡敌人池。
4. 不修改玩家技能树。

2. `test-results/campaign-balance-summary.md`

## 3. 自动试玩覆盖

自动试玩覆盖 30 个正式 story 关卡：

1. `level_1_1` 至 `level_1_10`
2. `level_2_1` 至 `level_2_10`
3. `level_3_1` 至 `level_3_10`

自动试玩覆盖 3 类玩家构筑：

1. `basic`：基础构筑。
2. `recommended`：推荐构筑。
3. `specialist`：偏科输出构筑。

每条结果记录包含：

1. 关卡 ID。
2. 玩家构筑。
3. 胜负。
4. 回合数。
5. 玩家剩余 HP。
6. 敌人剩余 HP。
7. 敌人技能使用分布。
8. 失败原因。
9. 平衡诊断标签。

## 5. 首轮报告结论

报告生成命令：

```bash
node tools/campaign_balance_simulator.mjs --report test-results/campaign-balance-report.json
```

报告摘要：

| 构筑 | 胜场 | 尝试 | 胜率 | 平均回合 | 平均玩家剩余 HP |
| --- | ---: | ---: | ---: | ---: | ---: |
| `basic` | 23 | 30 | 76.7% | 4.9 | 97.7 |
| `recommended` | 22 | 30 | 73.3% | 7.2 | 107.4 |
| `specialist` | 26 | 30 | 86.7% | 2.8 | 77.3 |

诊断标签计数：

| 标签 | 数量 |
| --- | ---: |
| `ok` | 40 |
| `enemy_too_weak` | 31 |
| `enemy_numbers_too_high` | 8 |
| `expected_basic_build_limit` | 5 |
| `player_build_mismatch` | 4 |
| `player_skill_tree_gap_candidate` | 2 |

当前主要结论：

1. 推荐构筑胜率为 73.3%，处在粗调下限附近。
2. 推荐构筑失败多数发生在玩家 HP 仍较高、敌人 HP 残留的场景，优先怀疑敌人 HP、护甲、修复或关卡敌人池数值偏高，而不是敌人技能压力过强。
3. 偏科输出构筑胜率为 86.7%，且平均回合数只有 2.8，说明纯输出泛用性偏高，后续需要让章节压力更有针对性。
4. `player_skill_tree_gap_candidate` 只出现在基础构筑普通关失败候选中，尚不足以证明需要补玩家技能树。

## 7. 后续任务

下一步进入 `WBS-3.4.5 调参与玩家技能树补洞`：

1. 优先检查推荐构筑失败但玩家 HP 仍较高的关卡：`level_1_4`、`level_1_10`、`level_2_4`、`level_2_9`、`level_2_10`、`level_3_1`、`level_3_3`、`level_3_10`。
2. 对过弱关卡补充压力，尤其是 1-2 回合被偏科构筑快速通过的关卡。
3. 先调整敌人数值、敌人技能和关卡敌人池，不直接修改玩家技能树。
4. 只有在自动试玩证明推荐构筑也缺少明确反制手段时，才进入玩家技能树补洞。
