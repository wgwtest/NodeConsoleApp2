# WBS-3.4.18 第二章护甲检查点压力调参与自测记录

## 背景

WBS-3.4.17 已补强浏览器“新学技能进入下一关并执行”的真实链路。本轮回到长期目标中的敌人难度调参：审计当前进度式 30 关报告后，发现 `level_2_3` 和 `level_2_8` 仍被诊断为 `enemy_too_weak`。

本轮不补玩家技能树。现有证据显示：

- 进度式模拟 30/30 通关。
- `skillTreeGapCandidates = 0`。
- 问题集中在第二章冰甲守卫检查点满血通过，属于敌人压力不足。

## 红灯

新增测试：

```bash
node --test test/campaign_balance_simulator.test.mjs
```

新增用例：`progressive campaign chapter two armor checkpoints create attrition`

红灯结果：

- `level_2_3: enemy_too_weak, 100/100`
- `level_2_8: enemy_too_weak, 100/100`

## 根因

两个关卡都复用 `enemy_c2_iceplate_guard / 冰甲守卫`。

调参前进度式结果：

- `level_2_3`
  - 3 回合胜利。
  - 玩家剩余 HP `100/100`。
  - 敌技使用：`enemy_skill_shield_tap = 1`，`enemy_skill_bone_patch = 3`。
- `level_2_8`
  - 3 回合胜利。
  - 玩家剩余 HP `100/100`。
  - 敌技使用：`enemy_skill_shield_tap = 1`，`enemy_skill_bone_patch = 3`。

判断：冰甲守卫作为第二章护甲/控制检查点，实战中过度偏向修甲，没有制造足够生命损耗。

## 改动

新增敌人技能：

- `enemy_skill_iceplate_head_crush / 冰盾压颅`
- 成本：`3 AP`
- 目标：玩家 `head`
- 效果：
  - `DMG_ARMOR ABS 18`
  - `DMG_HP ABS 18`
  - 75% 概率施加 `buff_slow`，持续 1 回合

更新敌人：

- `enemy_c2_iceplate_guard`
  - 标签新增 `pressure_slow`
  - 描述补充冰盾压制头部
  - 技能从通用 `enemy_skill_shield_tap` 切换为专属 `enemy_skill_iceplate_head_crush`

这样只抬高冰甲守卫，不影响其他继续使用通用骷髅盾击的关卡。
