# 技能新增 Codex 护栏与排布检查交接

创建时间：2026-05-22 11:28:07 +0800

## 1. 本轮目标

用户明确：技能新增约束不是给玩家、人工编辑者或 Web 技能编辑器使用，而是给 Codex 自己使用的生成前护栏。目的是避免 Codex 在协助新增技能时直接生成 JSON，导致节点堆叠、引用错误、成本不合理或误发布。

## 2. 已完成内容

1. 新增规程文档：
   - `DOC/CODEX_DOC/04_研发文档/18-技能新增Codex护栏与排布检查规程.md`
2. 新增检查器模块：
   - `test/support/skill_authoring_guard.mjs`
3. 新增测试：
   - `test/skill_authoring_guard.test.mjs`
4. 新增命令行检查入口：
   - `tools/validate_skill_authoring_guard.mjs`
5. 更新启动入口：
   - `START_HERE.md`
   - 后续新增/调整正式技能前必须先读护栏规程并运行检查器。

## 3. 关键约束

1. 不把“生成候选技能”做成技能编辑器核心能力。
2. Codex 可以生成 authoring 工作稿，但默认不能发布到 `assets/data/skills_melee_v4_5.json`。
3. 新增正式技能必须显式写入 `editorMeta.x/y`，不能依赖 fallback 布局。
4. 新增前必须先输出设计表，说明父节点、稀有度、KP、Buff 引用、复杂度和坐标。
5. 高复杂度 Buff/Skill 机制必须先单独分析，不能直接落 JSON。
