记录时间：2026-05-29 00:22:30 +0800

状态：已实现候选包与通用运行时能力，待用户审定候选技能设计

## 1. 本轮背景

用户明确要求剑系流血设计不能只是罗列“浅割、深切、反手割裂”等相近技能，而应建立 AP/KP/敌人数量约束下的理性模型。

本轮采用的模型：

```text
建窗 -> 维窗 -> 读窗 -> 消窗
```

其中 `W = buff_bleed.remaining`，流血每回合结束固定造成 5 点 HP 伤害，不按层数提高每回合伤害。

## 3. 重要实现事实

1. 候选包没有发布到正式运行时包。
2. 候选包以 `skills_melee_v4_5_sword_rework_20260528_004020.json` 为唯一源文件，保留 54 个技能、原坐标、原前置关系和非剑系内容。
3. `血涌斩` 使用 `maxRead: 4`，读取流血窗口但不消耗。
4. `断脉一剑` 和 `猩红收割` 使用 `consumeRemaining` 消耗流血窗口。
5. `targetBuff` 与 `targetHpPercentBelow` 释放条件现在由 CoreEngine 执行前通用校验，不再只是描述字段。
6. 规划 UI 是否提前禁用条件不足的技能还未做。

## 4. 纠偏记录

最初生成的 `skills_melee_v4_5_sword_bleed_window_v1_20260529_000000.json` 错误地基于运行版 `assets/data/skills_melee_v4_5.json` 派生，只有 45 个源技能，导致候选包与 `004020` 的 54 技能布局差距过大。该错误候选包已删除，不再作为评审对象。
