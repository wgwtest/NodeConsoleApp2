# WBS-3.4.6 进度式报告落盘与证据链自测记录

日期：2026-05-25

分支：`codex/enemy-design-20260522`

## 1. 本轮目标

补齐自动试玩证据链，让固定构筑报告和进度式成长报告都能通过 CLI 落盘，避免后续调参只依赖临时脚本输出。

本轮边界：

1. 不修改玩家技能树。
2. 不继续调敌人数值。
3. 只补报告入口、summary 渲染和回归测试。

## 2. 改动范围

文件：`tools/campaign_balance_simulator.mjs`

1. `writeCampaignBalanceReport()` 支持进度式报告。
2. summary 渲染识别 `meta.mode === "progressive"`。
3. 进度式 summary 输出：
   - 胜场 / 尝试次数 / 胜率
   - 技能树缺口候选数
   - 最终剩余 KP
   - 最终已学技能
   - 每关学习技能
   - 每关奖励 KP
4. CLI 新增：

```bash
node tools/campaign_balance_simulator.mjs --mode progressive --seed wbs-3.4.5-progressive-campaign --report test-results/campaign-progressive-report.json
```

5. 默认固定构筑报告仍保留旧路径：
   - `test-results/campaign-balance-report.json`
   - `test-results/campaign-balance-summary.md`

6. 非默认报告路径按 report 文件名派生 summary，避免覆盖：
   - `test-results/campaign-progressive-report.json`
   - `test-results/campaign-progressive-report-summary.md`
