# 技能编辑器保存发布与最近版本下拉交接

创建时间：2026-05-22 01:53:59 +0800

## 1. 本轮处理内容

本轮根据地图编辑器的保存 / 发布策略，补齐技能编辑器的工程化写回链路：

1. “保存工作稿”写入 `assets/skill_packs/authoring/`，并保留带时间戳的 JSON 版本。
2. “发布到主流程”写入 `assets/data/skills_melee_v4_5.json`，供主流程直接读取。
3. 工具栏新增“最近版本”下拉框，支持刷新并加载最近技能 JSON。
4. 加载项目技能时，继续自动加载同目录 Buff 包。

## 2. 关键实现

1. 后端新增技能包专用 API：
   - `GET /api/skill-packs/recent?limit=20`
   - `POST /api/skill-packs/save`
   - `POST /api/skill-packs/publish`
2. 保存工作稿限制在 `assets/skill_packs/authoring/`，避免误覆盖运行时数据。
3. 发布接口限制为 `assets/data/skills_melee_v4_5.json`，避免发布到主流程无法读取的位置。
4. 最近版本列表同时扫描 authoring 技能版本和 `assets/data/` 下的技能 JSON。
5. 连续保存工作稿时会去掉旧时间戳再生成新文件名，避免文件名不断膨胀。
