# WBS-3.4.26 终局 Boss 后期反制窗口与自然胜利记录

## 背景

WBS-3.4.25 已经把 `level_3_10` 的浏览器自然战斗失败诊断清楚：终局 Boss 不是打不动，而是末回合互杀后按玩家死亡结算为失败。本轮目标是在不新增技能、不降低终局 Boss 整体主题压力的前提下，让后期构筑形成可执行反制窗口，并让真实浏览器自然战斗可以低血胜利。

## TDD 记录

先更新 `test/campaign_balance_content.test.mjs`，要求后期吸血 `skill_execute_copy_1770044052832` 的有效速度早于暮堡重抡 `enemy_skill_twilight_bastion_swing`：

```bash
node --test test/campaign_balance_content.test.mjs
```

红灯原因：

- 后期吸血原速度为 `1`。
- 玩家基础速度 `1`，吸血有效速度为 `2`。
- 暮色壁垒基础速度 `7`，暮堡重抡速度 `-4`，有效速度为 `3`。
- 吸血晚于 Boss 致死爆发，低血反制窗口不可执行。

随后更新 `test/acceptance_click_smoke_contract.test.mjs`，要求自动点击脚本固定保留：

- `naturalLateGameProbeBuild`
- `lateGameNaturalProbeBuild`
- `terminalSurvivalFloor`
- `maintainTerminalBleedWindow`
- `terminalSustainRotation`
- `stopFillingTerminalPressureActions`
- `terminalBossNaturalOutcome`
- `skill_regroup`
- `skill_execute_copy_1770044052832`
- `mutual_kill_settlement_loss`

在第一次真实浏览器修正后，又补充合同测试：

- `terminalFinishWindow`
- `terminalDirectFinishBonus`
- `terminalNonDamageFinishPenalty`
- 残血窗口压低无直伤续航技能。
- 残血窗口提高可直接击杀技能优先级。

红灯结果确认：终局压力分支中原有残血收尾逻辑不可达，导致 Boss 剩余少量 HP 时仍反复选择吸血、治疗、格挡。

## 改动

更新 `assets/data/skills_melee_v4_5.json`：

- `skill_execute_copy_1770044052832` 后期吸血速度从 `1` 调整为 `3`。
- 玩家有效速度从 `2` 提升到 `4`，早于暮堡重抡有效速度 `3`。

更新 `tools/acceptance_click_smoke.mjs`：

- 新增 `naturalLateGameProbeBuild`，用于 `level_3_10` 的后期自然压力检查。
- `level_3_10` 改为：
  - `skillLoadoutSource = lateGameNaturalProbeBuild`
  - `requireVictory = true`
  - `requirePressure = true`
- 保留自然战斗失败诊断字段，继续识别 `mutual_kill_settlement_loss`。
- 增加终局压力策略：
  - 低血时优先保留 `skill_regroup`、`skill_block`、吸血和流血维护窗口。
  - 在 Boss 残血且玩家不低于危险线时触发 `terminalFinishWindow`。
  - 对不造成直接伤害的续航技能施加 `terminalNonDamageFinishPenalty`。
  - 对可直接击杀的技能施加 `terminalDirectFinishBonus`。
