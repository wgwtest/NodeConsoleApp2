# WBS-3.4.14 浏览器多章节关卡选择 Smoke 记录

## 背景

WBS-3.4.13 已补齐 30 个 story 关卡的运行时首回合 smoke，但浏览器自动点击仍只实际进入 `level_1_1`。本轮补强浏览器层关卡选择覆盖：在真实 DOM 中切换三张章节地图，并分别选中三章 Boss 关，验证地图切换、节点选中和详情抽屉进入按钮的基础链路。

## 根因确认

主流程实际加载的是目录式地图包：

- `assets/map_packs/current/story_pack_v1/maps.json`

因此生产 DOM 中的地图 ID 为：

- `chapter_1_authoring_map`
- `chapter_2_authoring_map`
- `chapter_3_authoring_map`

不是旧兼容包 `assets/data/level_map_pack_v1.json` 中的 `chapter_1_story_map` 等 ID。自动化脚本已绑定真实 DOM 的 `data-map-id` 和 `data-level-id`。

## 改动

更新 `tools/acceptance_click_smoke.mjs`：

- 新增 `runLevelSelectMultiMapSmoke(cdpEndpoint, mainUrl)`。
- 在主报告中新增 `levelSelectMultiMap` 字段。
- 验证三张地图切换按钮存在。
- 依次切换并选中：
  - `chapter_1_authoring_map` / `level_1_10`
  - `chapter_2_authoring_map` / `level_2_10`
  - `chapter_3_authoring_map` / `level_3_10`
- 只验证选中和详情抽屉，不反复进入战斗，避免一个 smoke 同时承担多场战斗执行。

更新 `test/acceptance_click_smoke_contract.test.mjs`：

- 契约固定新增函数名和三章地图 / 三章 Boss 关 ID。

## 实际浏览器验收

启动环境：

```bash
PORT=3111 node app.js
google-chrome --headless=new --disable-gpu --no-first-run --no-default-browser-check \
  --remote-debugging-address=127.0.0.1 --remote-debugging-port=9223 \
  --user-data-dir=/tmp/nodeconsoleapp2-cdp-enemy-9223 about:blank
```

验收命令：

```bash
APP_BASE_URL=http://127.0.0.1:3111 CDP_ENDPOINT=http://127.0.0.1:9223 node tools/acceptance_click_smoke.mjs
```

关键结果：

- 地图切换按钮：
  - `第一章 幽暗森林 12/12`
  - `第二章 霜雾峡谷 12/12`
  - `第三章 暮色古道 12/12`
- 选中结果：
  - `level_1_10`：`♛ 1-10`，详情标题 `废墟关隘`
  - `level_2_10`：`♛ 2-10`，详情标题 `冰封中庭`
  - `level_3_10`：`♛ 3-10`，详情标题 `暮堡外环`
- `finalState = MAIN_MENU`，说明多地图选择 smoke 没有误进入战斗。
- 本轮新增流程无 Runtime exception。

## 当前判断

浏览器层现在覆盖两类路径：

1. 主流程进入 `level_1_1` 并执行一回合。
2. 关卡选择页跨三章切图并选中三章 Boss 关。

这与 WBS-3.4.13 的 30 关运行时 smoke 组合后，可以证明“浏览器地图选择基础交互”和“30 关战斗首回合运行时链路”都没有基础断链。后续如果继续增强实际体验验收，应再补胜利结算、技能学习后下一关进入、战斗 UI 状态反馈截图审计。
