# WBS-3.4.12 多种子平衡稳定性审计与调参止点记录

## 背景

WBS-3.4.11 后，固定报告只剩 2 条 `enemy_too_weak`，进度式报告仍为 30/30。此时继续追求 `enemy_too_weak = 0` 有较高风险：可能把章节中段的节奏缓冲关全部调成高压关。

本轮目标不是继续调数值，而是用多种子自动试玩确认当前结果是否稳定，并判断是否应该保留 `level_2_6` 一类缓冲关。

## 审计命令

```bash
node - <<'NODE'
import { runCampaignBalanceSimulation, runProgressiveCampaignSimulation } from './tools/campaign_balance_simulator.mjs';
const seeds = ['campaign-balance-v1','wbs-3.4.11-remaining-weak-concentration','balance-a','balance-b','balance-c','balance-d','balance-e'];
for (const seed of seeds) {
  const fixed = await runCampaignBalanceSimulation({ projectRoot: process.cwd(), maxTurns: 12, randomSeed: seed });
  const prog = await runProgressiveCampaignSimulation({ projectRoot: process.cwd(), maxTurns: 12, randomSeed: seed });
  const rec = fixed.summary.byBuild.find(row => row.buildId === 'recommended');
  const spec = fixed.summary.byBuild.find(row => row.buildId === 'specialist');
  const basic = fixed.summary.byBuild.find(row => row.buildId === 'basic');
  console.log(seed, {
    fixedWins: [basic.wins, rec.wins, spec.wins],
    fixedWeak: fixed.summary.diagnosisCounts.enemy_too_weak || 0,
    fixedPressure: fixed.summary.diagnosisCounts.enemy_skill_pressure_high || 0,
    progressiveWins: prog.summary.wins,
    skillTreeGapCandidates: prog.summary.skillTreeGapCandidates,
    recAvgHp: rec.averagePlayerRemainingHp,
    specAvgHp: spec.averagePlayerRemainingHp
  });
}
NODE
```

跨 seed 稳定结论：

- 固定三构筑均稳定为 27/30。
- recommended 固定失败稳定集中在：
  - `level_2_10`
  - `level_3_9`
  - `level_3_10`
- 进度式构筑稳定为 30/30。
- `Skill tree gap candidates` 稳定为 0。
- fixed `enemy_too_weak` 在 0-2 条之间波动，没有继续集中在同一个敌人或同一个机制。
- fixed `enemy_skill_pressure_high` 稳定为 3，说明 Boss / 精英压力仍保留。

## 决策

本轮不继续调参。

理由：

1. 当前已经满足三章主线进度式可通关。
2. 固定推荐构筑仍有 3 个预期高压失败点，没有变成全胜碾压。
3. 偏科构筑平均剩余 HP 明显低于推荐构筑，说明敌人压力仍能惩罚错误构筑。
4. 剩余 `enemy_too_weak` 数量低，且跨 seed 不稳定集中，不构成明确内容缺陷。
5. 继续追零会减少章节节奏缓冲，反而可能让 30 关体验过于绷紧。

## 后续建议

- 暂不补玩家技能树：没有技能树缺口证据。
- 暂不继续追求 `enemy_too_weak = 0`。
- 下一阶段更适合从“可玩”转向“体验质量”：
  - 人工试玩或自动点击走完整章节流程。
  - 检查战斗 UI 是否清楚呈现敌人技能与状态。
  - 观察玩家是否能理解第二章护甲 / 修复主题和第三章毒性主题。
  - 若要继续数值调参，应以人工试玩痛点或新报告异常为入口，而不是只看单个弱关标签。
