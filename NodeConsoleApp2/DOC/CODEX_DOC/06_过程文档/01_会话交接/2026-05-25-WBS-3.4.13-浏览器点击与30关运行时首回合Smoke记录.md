# WBS-3.4.13 浏览器点击与 30 关运行时首回合 Smoke 记录

## 背景

WBS-3.4.12 已确认多种子平衡结果稳定，且暂不继续追求 `enemy_too_weak = 0`。本轮目标转向实际流程测试：先用浏览器自动点击验证主 UI 可进入战斗并执行回合，再补一层 30 个 story 关卡的运行时首回合 smoke，避免只验证 `level_1_1`。

## 浏览器自动点击

启动环境：

```bash
PORT=3111 node app.js
google-chrome --headless=new --disable-gpu --no-first-run --no-default-browser-check \
  --remote-debugging-address=127.0.0.1 --remote-debugging-port=9223 \
  --user-data-dir=/tmp/nodeconsoleapp2-cdp-enemy-9223 about:blank
```

验收命令：

```bash
APP_BASE_URL=http://127.0.0.1:3111 CDP_ENDPOINT=http://127.0.0.1:9223 node tools/acceptance_click_smoke.mjs
```

结果摘要：

- 主流程打开 `mock_ui_v11.html`。
- 自动点击 `新游戏` -> `关卡选择` -> 选择 `level_1_1` -> 进入战斗。
- 战斗内完成技能选择、落槽、`提交规划`、`执行回合`。
- 执行前 `执行回合` 按钮确认为禁用，提示为“请先提交规划，形成可执行时间轴。”
- 执行后 `currentTurn = 2`，`timelineAfterExecute = FINISHED`，`presentationSource = asset`。
- `battle_presentation_probe.html` 和 `battle_presentation_configurator.html` 均完成自动点击，无 Runtime exception。

## 30 关运行时首回合 Smoke

新增入口：

- `tools/campaign_balance_simulator.mjs`
  - `runCampaignRuntimeSmoke(options)`

新增测试：

- `test/campaign_balance_simulator.test.mjs`
  - `campaign runtime smoke enters all 30 story levels and executes the first combat round`

报告生成命令：

```bash
node --input-type=module -e "import { runCampaignRuntimeSmoke, writeCampaignBalanceReport } from './tools/campaign_balance_simulator.mjs'; const report = await runCampaignRuntimeSmoke({ projectRoot: process.cwd(), randomSeed: 'wbs-3.4.13-runtime-smoke' }); await writeCampaignBalanceReport(report, { reportPath: 'test-results/campaign-runtime-smoke-report.json' }); console.log(JSON.stringify({ meta: report.meta, summary: report.summary }, null, 2));"
```

报告产物：

- `test-results/campaign-runtime-smoke-report.json`
- `test-results/campaign-runtime-smoke-report-summary.md`

结果摘要：

- `coveredLevels = 30`
- `failedLevels = []`
- 30 个 story 关卡均能：
  - 实例化至少 1 个敌人。
  - 生成首回合玩家规划。
  - 生成首回合敌方计划。
  - 构建包含敌我行动的时间轴。
  - 执行首回合，并推进到第 2 回合或进入结算。

## 当前判断

本轮证据比原浏览器点击 smoke 更完整：浏览器层证明主 UI 至少可真实点击进入战斗并执行回合；运行时层证明 30 个正式关卡都能通过 `CoreEngine` 首回合链路，不存在敌人实例化、敌方技能规划或时间轴构建的基础断链。

这仍不等于“完整人工体验验收完成”。后续如果继续推进最终目标，下一层更应该补：

- 多关浏览器路径覆盖，而不是只点 `level_1_1`。
- 战斗 UI 中敌方技能、buff、护甲破损、治疗/修复反馈的可读性截图审计。
- 章节切换、胜利结算、技能学习后的下一关进入流程。
