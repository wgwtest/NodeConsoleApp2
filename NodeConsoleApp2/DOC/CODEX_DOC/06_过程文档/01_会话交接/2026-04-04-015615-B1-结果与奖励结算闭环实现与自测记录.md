## 2. 关键代码变更

### 2.1 DataManager 结算与归一化

`script/engine/DataManagerV2.js`

1. 新增 `_normalizePlayerResources()`
2. 新增 `_normalizeProgress()`
3. 新增 `_normalizeBattleRewards()`
4. 新增 `getLevelRewards(levelId)`
5. 新增 `applyBattleSettlement({ levelId, victory })`

当前结算会更新：

1. `player.resources.exp`
2. `player.resources.gold`
3. `player.skills.skillPoints`
4. `global.progress.completedLevels`
5. `global.progress.lastSettlement`

### 2.2 CoreEngine 结算事件收口

`script/engine/CoreEngine.js`

1. `endBattle()` 现在会先基于当前关卡生成 settlement
2. `BATTLE_SETTLEMENT` 状态参数中会带 `settlement`
3. `BATTLE_END` 事件中也会带 `settlement`
4. 结算后会保存主菜单场景下的新存档
5. `_buildBattlePlayerState()` 现在会同时构建 `stats / bodyParts` 运行时覆盖
6. `startBattle()` 会先把关卡注入的玩家战斗数值同步到运行时，再发出 `BATTLE_START`

### 2.3 UI 结算展示升级

4. 首次通关时会显示提示
5. 按钮文案统一为 `确认并返回主菜单`
