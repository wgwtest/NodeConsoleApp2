# mock_ui_v11 主流程测试基线

创建时间：2026-03-28 00:01

## 1. 目的

本文件用于固化 `mock_ui_v11.html` 的主流程测试入口和典型点击流程。

该流程由用户明确给出，后续所有基于 `mock_ui_v11.html` 的主流程自测、验收和回归，默认都应以本文件为依据，而不是由 Codex 自行推断页面意图。

## 2. 主测试入口

主入口：

- `http://127.0.0.1:3101/mock_ui_v11.html`

适用范围：

- 主流程 UI 验收
- 多回合战斗流程验证
- 技能树学习后的战斗回归
- 规划与执行链路验证

辅助入口：

- `test/engine_test_v2.html`

辅助入口只用于：

- 引擎底层行为隔离验证
- 快速对照验证
- 当主 UI 干扰判断时的运行时剥离验证

## 3. 用户确认的典型点击流程

### 3.1 主流程

标准步骤如下：

1. 进入页面后，先点击 `开始冒险`
2. 弹出 `游戏菜单` 后，点击 `关卡选择`
3. 选择关卡
   - 当前只有一个关卡，直接选这一个
4. 进入游戏主界面后，默认就是第一回合
5. 点击技能按钮
6. 将技能部署到技能槽
   - 槽位可随机选，用于测试
7. 点击 `提交规划`
8. 点击 `执行回合`
9. 后续自动执行
10. 进入下一回合

## 4. 多轮测试补充说明

用户建议：

- 在多轮测试时，可以执行 `技能树 / 构筑`
- 在技能树模态窗中，可以学习一定技能后再回到主流程继续测试

这说明：

- 技能树属于主流程关联子流程
- 不是必须每轮都测
- 但在多轮回归中应纳入覆盖

## 5. 技能树子流程

当前已知规则如下：

1. 点击 `技能树 / 构筑`
2. 在技能树模态窗中选择技能学习
3. 点击 `提交并关闭` 可以学习技能并关闭模态窗
4. 点击 `×` 只关闭模态窗，不应等同于学习成功

后续测试要求：

- 学习技能后应验证技能是否真的进入可用技能列表
- 不应只验证模态窗关闭

## 6. 当前不纳入主流程的内容

以下内容用户已明确说明当前不重要，默认不纳入主流程验收：

- 时间线区域的：
  - `开始`
  - `单步`
  - `直接结算`
- 游戏过程中的设置按钮
- 齿轮按钮进入的设置流程

这意味着：

- 上述区域可以保留为后续扩展测试项
- 但当前不应把它们当成主流程阻塞项

## 7. 当前测试约束

后续基于 `mock_ui_v11.html` 做测试时，应遵守以下约束：

- 优先用页面真实点击流程，不先走脚本直改运行态
- 只有在验证底层状态、定位问题或构造对照组时，才直接注入运行态
- 主流程报告必须区分：
  - UI 点击流程结论
  - 运行态注入验证结论

## 8. 对后续文档的要求

从本文件开始，后续所有相关文档都应明确标注：

1. 是否按本文件的主流程执行
2. 是否包含技能树子流程
3. 是否用了运行态注入作为辅助验证
4. 哪些发现属于主 UI 问题
5. 哪些发现属于引擎运行时问题
