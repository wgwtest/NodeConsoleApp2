# WBS-3.2.3 计划：地图编辑器基础能力与地图包导出链

创建时间：2026-04-12

最后整理时间：2026-04-12

状态：`当前有效`

## 1. 节点定位

本节点属于：

1. `WBS-3.2 关卡与地图编辑`
2. `#101 WBS-3.2.3 地图编辑器基础能力与地图包导出链`

它与 `#79 / WBS-3.2.2 关卡编辑器基础能力与 JSON 导出链` 的关系是：

1. `#79 / WBS-3.2.2`
   - 维护关卡侧运行时配置
2. `#101 / WBS-3.2.3`
   - 维护地图侧展示与组织配置

它下挂三个正式子节点：

1. `#100 / WBS-3.2.3.1 地图组织方案与效果图验证`
2. `#102 / WBS-3.2.3.2 地图编辑工作区与节点边背景编辑`
3. `#103 / WBS-3.2.3.3 地图包导入导出与校验`

## 2. 本节点为什么要独立存在

如果地图编辑器不独立出来，而是继续混在总树或方案页中，会产生三个问题：

1. 看树时无法判断“地图编辑”是否已经是正式实现节点
2. 地图背景、节点、边、样式与地图包导出会被误解成“只是一个演示页”
3. 关卡编辑器与地图编辑器的边界会再次被写混

因此本节点必须以独立主树存在。

## 3. 本节点目标

本节点最终应交付：

1. `地图节点 CRUD`
2. `节点拖拽定位`
3. `背景切换`
4. `边关系编辑`
5. `节点样式编辑`
6. `level_map_pack` 导入 / 导出
7. 地图侧强校验
