## 2. 为什么阶段 D 现在就要启动 D3

虽然当前正式阶段已经切到 `#28 / 阶段C`，但 `D3` 仍属于典型低耦合 Prep：

1. 它只做展示层
2. 它不改伤害、Buff、敌人决策和存档
3. 它可以独立在 probe 页验证
4. 它可以提前暴露“表现层与规则层是否真正解耦”的风险

因此本节点可以在阶段 D `Core` 尚未正式开始前，先按 `Prep` 推进。

## 3. 本节点目标

`D3` 的目标不是单纯“加动画”，而是让战斗表现层满足交付态的基本要求：

1. 关键动作有可观察反馈
2. 反馈来源于稳定事件，而不是写死在页面按钮逻辑里
3. 表现层能独立验证
4. 表现层失败不影响主流程

## 4. 当前切片策略

本轮只落地最小闭环：

1. 先做纯展示层动画驱动
2. 先做独立 probe 页
3. 先做一个角色的轻量纸娃娃试验
4. 再把同一驱动最小接入 `mock_ui_v11.html`
5. 不在本轮扩成完整美术表现系统

## 7. 验收口径

本节点不是在验收“战斗规则”，而是在验收“画面是否把规则结果表达清楚”。

必须能回答：

1. 为什么这个展示层是低耦合的
2. 为什么 probe 页能证明展示层独立成立
3. 为什么主流程页能证明动画没有阻断真实游玩
4. 为什么本轮轻量纸娃娃试验不会把系统绑死在单一实现上
