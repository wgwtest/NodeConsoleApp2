## 2. 为什么阶段 C 需要单独的内容扩充包

阶段 C 不能只做“成长入口”和“技能树入口”。

如果没有可感知的内容差异，玩家即使拿到了成长资源，也无法明确感受到：

1. 不同关卡在打什么
2. 不同敌人风格有什么差别
3. 奖励节奏是否值得冒险
4. 当前构筑应该往哪里准备

因此 `C2` 的职责不是简单加数量，而是逐步把“内容差异”做成玩家能看见、能理解、能验证的正式体验。

## 7. 验收口径

本节点不是在验证“能不能进入关卡”，而是在验证“玩家进入前能不能看懂关卡差异”。

必须能回答：

1. 三个 story 关卡为什么不是同一张卡片换名字
2. 玩家能不能在进入前看见敌人风格
3. 玩家能不能在进入前看见奖励节奏
4. 新信息是否会破坏原有解锁态 / 已完成态
