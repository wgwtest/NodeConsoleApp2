## 3. 责任边界

本节点负责：

1. 导出文件名、导出内容与契约字段
2. override 的写入与清理
3. 通过 `level_runtime_probe.html` 验证运行时重新消费

2. 重新组织正式编辑工作区布局

## 4. 当前建议实现切片

建议本叶子优先形成三条可复用验证链：

1. `编辑器 -> 导入 JSON -> 编辑 -> 导出 JSON -> 下载`
2. `编辑器 -> override -> level_runtime_probe`
3. `静态语义回归 -> 回归页摘要 -> 主流程 smoke`

同时需要补齐：

1. 至少一个合法样例
2. 至少一个非法场景的校验反馈
3. 导出后再装载的 round-trip 一致性检查
4. `unlockRules` / `victoryCondition` / `failureCondition` 在 probe 页中的可见证据

## 5. 验收标准

通过标准至少包括：

1. `level_editor_v1.html` 可以导入现有 JSON、导出并写入 override
2. `level_runtime_probe.html` 可以独立证明运行时已消费 override，并显示 `unlockRules` / `victoryCondition` / `failureCondition`
3. `level_editor_io_test.html` 与 `codex_regression_runner.html` 不再冒充正式编辑器，而是明确承担 IO / 回归职责
4. `level_editor_export.contract.json` 能为 `WBS-3.2.3` 提供稳定输入
5. `tools/level_editor_cdp_smoke.mjs` 能在真实浏览器里验证整条闭环
