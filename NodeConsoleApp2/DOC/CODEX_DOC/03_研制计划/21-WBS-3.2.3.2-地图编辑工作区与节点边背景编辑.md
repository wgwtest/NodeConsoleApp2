## 2. 本节点职责

本节点只负责地图编辑工作区本体，不负责地图包 IO 校验。

应交付：

1. 地图节点增删改
2. 基于逻辑地图坐标的数据驱动定位
3. 边关系编辑
4. 背景切换
5. 节点样式编辑
6. 背景图 / 节点素材图 / 立绘图的真实图片资源消费链

当前补强重点：

1. 地图真值必须先落在地图数据对象，而不是落在 DOM 或抽屉状态上
2. 节点位置必须表达“逻辑地图坐标”，不能表达“当前 div 左上角像素”
3. 边只保存节点关系，不保存起点终点的页面像素端点
4. 背景图、节点素材图、立绘图先写入数据，再由数据显示层消费
5. 抽屉开关、画布缩放、视口变化只能改变投影结果，不能改变地图真值
6. `space.logicalWidth / logicalHeight` 与 `display.viewportAspect / backgroundFit / nodeScale / edgeLabelMode / nodeAnchor / edgeAnchor` 应提供可见编辑入口，并同步影响显示层与导出结果
7. 作者工具摘要应能同步说明地图编辑页当前已具备的关键编辑能力，避免系统菜单与工具页状态脱节

## 4. 验收标准

1. 可以新增、删除和移动地图节点
2. 节点位置来源于逻辑地图坐标，而不是当前 DOM 像素
3. 可以编辑边关系、背景和节点样式
4. 工作区交互不直接侵入战斗主流程
5. 页面会明确解释本页用途、按钮原因与 `level_map_selection_mock.html（历史验证资产） / level_editor_v1.html / WBS-3.2.3.3` 的关系
6. 共享回归页已可通过 `scope=WBS-3.2.3.2` 辅助检查入口完整性
7. 编辑页必须真实消费 `assetLibrary.backgrounds[].src / nodeArts[].src / portraits[].src`
8. 右侧检查器必须提供背景图、节点素材图、立绘图的可见配置入口与预览
9. 抽屉开关、视口变化或主舞台缩放时，节点相对地图的位置关系不得漂移
10. 边的导出数据只表达 `fromNodeId / toNodeId / type / branchLabel`，不表达像素端点
11. 页面必须明确说明“地图真值”和“显示配置”的区别，避免把 DOM 像素当成地图数据
12. 修改 `logicalWidth / logicalHeight / viewportAspect / backgroundFit / nodeScale / edgeLabelMode / nodeAnchor / edgeAnchor` 后，中央舞台与导出预览必须同步更新

## 4.1 2026-04-12 远端评审纳入项

来源：

1. `#102 / WBS-3.2.3.2` GitHub 评论
2. 评论来源的 GitHub `author_association` 为 `NONE`
3. 本轮因用户明确要求“拉取远端评论，并分析，更新”，视为已获得用户确认，可纳入正式处理

评审项拆解与处理口径：

1. `中央画布不像最终输出，而像原型/调试视图`
   - 判断：`成立`
   - 说明：当前编辑页虽然已具备工作区能力，但中央画布仍保留过多内部说明块与调试式信息，离“后续主流程里地图展开描述的依据”还有距离
   - 处理：后续应把 `Map ID / Owner / 说明` 等内部元信息降级到次级区域，把中央画布提升为更接近正式地图表现的主舞台
2. `如果 3.2.3.2 只是编辑版，则缺少直观预览页与所见即所得关系`
   - 判断：`已吸收`
   - 说明：当前编辑页已承担主要所见即所得职责；`#100 / WBS-3.2.3.1` 的 `test/level_map_selection_mock.html` 保留为历史验证资产，不再作为编辑页强入口
   - 处理：已移除“打开剧情预览页”强按钮，改为明确说明该页仅作归档验证使用
3. `节点与边的绘制效果过差`
   - 判断：`成立`
   - 说明：即使已经完成一轮边标签与检查器改造，当前节点/边仍明显偏向作者工具草图，而非可交付的地图视觉母版
   - 处理：后续应把节点容器、边线、分支标签和场景氛围作为独立视觉层推进，而不是继续只做表单驱动页
4. `背景图层与节点显示效果不足，节点不能全部只是框`
   - 判断：`成立`
   - 说明：如果只是保留 `backgroundRef / artRefs.*` 字段，但页面仍使用程序生成的占位图，就不构成真正的素材化能力
   - 处理：当前已改为真实图片资源路径消费，样例资产落在 `assets/images/level_map/`，中央舞台与检查器都可见
5. `UI skill 清单要给具体候选，而不是抽象建议`
   - 判断：`成立`
   - 说明：后续需要给出可选 skill 列表，供用户选定统一的前端设计辅助路径
