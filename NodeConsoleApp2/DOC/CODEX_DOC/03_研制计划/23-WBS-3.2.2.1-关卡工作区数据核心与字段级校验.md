## 3. 责任边界

本节点负责：

1. `levels_v1_wrapped` 的内部真值组织
2. 关卡 id、章节信息、波次与敌人池引用的完整性
3. 关卡增删改后的关系修剪与排序稳定性
4. 后续页面层可直接消费的读写 API

本节点不负责：

1. 页面交互控件长什么样
2. 用户点击按钮后如何下载 JSON
3. 运行时如何从 override 中重新装载

## 4. 当前建议实现切片

建议本叶子优先收口这些能力：

1. `story / acceptance` 排序规则与章节内顺序
2. 删除关卡时的相关数据修剪与敌人池引用清理
3. 缺失敌人池等结构错误的统一校验输出
4. 为后续 `victory / failure / unlockRules` 等字段预留校验插槽，而不是直接耦到页面里

## 5. 验收标准

通过标准至少包括：

1. `node --test test/level_editor_workspace.test.mjs` 可覆盖核心读写与校验行为
2. 页面层不必知道底层文档细节，也能通过工作区 API 读写关卡
3. 当前字段级校验输出足以支撑 `3.2.2.2` 和 `3.2.2.3` 的后续实现
