# 地图编辑器三章 Authoring 地图实施计划

## 目标

把三章地图制作重心转到地图编辑器：编辑器默认加载三张可编辑 authoring 地图，每张不少于 10 个节点；主流程生产地图包暂不替换，等地图在编辑器中调满意后再同步。

## 非目标

- 本轮不把 `assets/data/level_map_pack_v1.json` 升级为新三章地图。
- 本轮不改战斗运行规则。
- 本轮不做文件系统自动保存；编辑器继续通过导出 JSON 交付地图包。

## 文件范围

- 新增：`assets/data/level_map_pack_v1.authoring.json`
- 修改：`test/level_map_editor_v1.html`
- 修改：`script/editor/level/LevelMapWorkspace.js`
- 修改：`script/editor/level/LevelMapEditorPage.js`
- 修改：`test/level_map_editor_page.test.mjs`
- 可能修改：`test/level_map_pack_io.test.mjs`

## 实施步骤

1. 补测试：确认 authoring 地图包存在、包含三张地图、每张不少于 10 个节点、结构校验无问题。
2. 补测试：确认编辑器默认加载 `level_map_pack_v1.authoring.json`，而不是示例包。
3. 补测试：确认编辑器节点 DOM 提供类型符号字段，类型识别不依赖颜色。
4. 新增 authoring 地图包：从 v1 蓝图调整为更适合编辑器微调的三章地图数据。
5. 修改编辑器默认数据源和页面说明：明确这是 authoring 工作地图包，导出后再同步到运行态。
6. 修改编辑器节点渲染：保留圆形节点图标，弱化标签，增加符号 badge。
7. 运行地图相关测试，必要时修正数据或投影问题。
8. 启动本地服务，打开编辑器做截图/DOM 自检。

## 验收方法

- `node --test test/level_map_pack_io.test.mjs test/level_map_workspace.test.mjs test/level_map_editor_page.test.mjs test/level_map_preview_page.test.mjs test/level_select_map_runtime.test.mjs test/level_select_map_view.test.mjs`
- 编辑器入口：`http://127.0.0.1:3121/test/level_map_editor_v1.html`
- 人工检查：地图列表出现三章；每章节点数量不少于 10；节点可拖拽；边可编辑；导出 JSON 包含三章 authoring 地图。
