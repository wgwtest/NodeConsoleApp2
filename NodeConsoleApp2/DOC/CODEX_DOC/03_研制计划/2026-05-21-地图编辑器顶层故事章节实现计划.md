# 地图编辑器顶层故事章节实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把地图编辑器从 flat `maps[]` 工作区推进到第一版“故事 -> 章节 -> 地图”的可用编辑闭环，并提供目录式地图包入口给主游戏加载。

**Architecture:** 数据真值放在 `LevelMapWorkspace`，页面只调用工作区 API 渲染和触发操作。运行时仍消费规范化后的 `level_map_pack_v1` 地图模型，但允许 `config.json` 指向目录式包入口 `package.json`，再读取 `maps.json`。

**Tech Stack:** 原生 ES module、Node `node:test`、JSDOM、当前静态资源服务。

---

## File Structure

1. `script/editor/level/LevelMapWorkspace.js`
   - 规范化 `stories[] / chapters[] / maps[]`。
   - 兼容旧 flat `maps[]` 自动放入默认故事和章节。
   - 提供故事、章节、地图级管理 API。
   - 导出目录式包 manifest 和 maps 文档。
2. `script/editor/level/LevelMapPackIO.js`
   - 汇总故事、章节、地图和资源依赖。
   - 输出 package manifest 预览。
3. `script/editor/level/LevelMapEditorPage.js`
   - 渲染故事/章节/地图树。
   - 绑定新建故事、新建章节、新建/复制/删除地图。
   - 导出弹窗展示目录式地图包结构。
4. `test/level_map_editor_v1.html`
   - 增加故事、章节、地图级按钮和输入。
   - 明确完整包入口文件。
5. `script/engine/DataManagerV2.js`
   - 支持 `levelMapPack` registry 使用 `packagePath` 或 `packageJsonPath`。
   - 读取 `package.json` 后加载 `maps.json`。
6. `assets/data/config.json`
   - 保持默认直接 JSON 路径，避免这次破坏主流程。
7. `assets/map_packs/current/story_pack_v1/package.json`
   - 新增目录式地图包入口样例。
8. `assets/map_packs/current/story_pack_v1/maps.json`
   - 新增目录式地图包数据样例。
9. `assets/map_packs/current/story_pack_v1/asset-manifest.json`
   - 新增目录式地图包资源依赖清单样例。
10. Tests
   - `test/level_map_workspace.test.mjs`
   - `test/level_map_pack_io.test.mjs`
   - `test/level_map_editor_page.test.mjs`
   - `test/level_select_map_runtime.test.mjs`
