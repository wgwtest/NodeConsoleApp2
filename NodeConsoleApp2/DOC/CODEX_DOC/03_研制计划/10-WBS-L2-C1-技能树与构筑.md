## 2. 为什么阶段 C 需要单独的技能树与构筑包

`C1` 的目标不是单纯把技能树做成“能打开的界面”。

如果玩家只能打开技能树，却感知不到：

1. 成长资源从哪里来
2. 最近学习改变了什么
3. 当前构筑和上一轮相比有什么差异
4. 这些差异如何真实影响下一局

那么技能树仍然只是研发内部工具，而不是正式游戏体验的一部分。

因此 `C1` 需要单独作为阶段 C 的稳定工作包，持续把：

1. 成长来源
2. 学习结果
3. 构筑差异
4. 下一局反馈

收口成玩家可感知、可验证、可追踪的正式闭环。
