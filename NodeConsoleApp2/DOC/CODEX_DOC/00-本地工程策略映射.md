   - `Contributes To`
   - `Work Type`
   - `Parent issue`
4. 解释规则：

   - `Parent issue` 承担 `Parent Node` 语义

### 2.2 Execution Contract

当前工程的 `Execution Contract` 为 GitHub Issue。

当前工程固定使用以下 10 项作为默认执行模板：

1. `目标`
2. `Owner`
3. `Write Scope`
4. `Depends On`
5. `验收入口`
6. `验收标准`
7. `输出物`
8. `证据链接`
9. `当前状态`
10. `上级节点`

补充规则：

1. GitHub Issue 正文只使用仓库相对路径，不使用 `/home/...` 绝对路径。
2. 当本地文档目录或文件名发生变化时，必须在同一轮同步修正 Issue / Project 中的路径引用。
3. 当用户明确验收通过某个 WBS 节点后，应在同一轮同步更新：
   - 本地验收清单状态
   - GitHub Issue 正文中的 `当前状态`
   - GitHub Project 中对应条目的粗粒度状态
